## システムアーキテクチャ概要

このrepoはRenCrow workspace内のCore / Chat / Viewer / Server正本である。`RenCrow_CMD`は同じ`/viewer/*` HTTP APIを叩くCLI facadeであり、仕様・挙動・責務判断はこのrepoを主とする。

主要層は次の通り。

| 層 | 主な場所 | 役割 |
|---|---|---|
| Composition root | `cmd/picoclaw`, `cmd/picoclaw-agent` | server/agent起動、config、runtime dependency、route登録 |
| Adapter | `internal/adapter/*`, `modules/*` | HTTP/Viewer/channel/module bridge |
| Application | `internal/application/*` | orchestrator、IdleChat、knowledge、job、verification、feature use case |
| Domain | `internal/domain/*` | agent/routing/task/session/memory/job/execution等の中核契約 |
| Infrastructure | `internal/infrastructure/*` | LLM/STT/TTS/MCP/webgather/tool runner/security/store/provider |
| Docs/config/data | `docs`, `config`, `prompts`, `skills`, `schemas`, `rencrow-data` | 正本仕様、runtime設定、prompt、知識・運用データ |

## 主要データフロー

### 会話

```text
Viewer/channel input
  -> adapter handler
  -> MessageOrchestrator / DistributedOrchestrator
  -> routing category
  -> Mio/Shiro/Coder/Heavy/Wild
  -> LLM provider / tools / memory
  -> response + events + evidence/logs
```

### Viewer運用監視

```text
orchestrator/background/store events
  -> EventHub.OnEvent
  -> MonitorStore reducers
  -> /viewer/events + /viewer/* status APIs
  -> browser tabs
```

### 記憶・知識

```text
session/user/wiki/source registry
  -> persistence/conversation L1 stores
  -> application knowledge/memory services
  -> Viewer memory APIs / orchestrator context
```

### 実行・検証

```text
Coder/Worker proposal or tool request
  -> application orchestrator/toolloop
  -> infrastructure ToolRunner + security policy
  -> execution/evidence/verification persistence
  -> Viewer evidence/job/verification APIs
```
