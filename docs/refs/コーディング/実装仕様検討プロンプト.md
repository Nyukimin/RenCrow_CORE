# 作成するドキュメント名

# 前提仕様

既存仕様 `coding_agent_modes.md` では、RenCrow のコーディングAIを以下の2形態に分けている。

1. Safe Build Mode
   既存システムを壊さずに変更を積み上げるモード。
   既存コード、既存DB、既存環境、設定、運用系に触る場合に使う。

2. Tool Build Mode
   れんが作ってほしい道具を素早く形にするモード。
   新規ツール、小物スクリプト、補助アプリ、検証用CLIなどを作る場合に使う。

また、Worker / Coder はモードそのものではなく、各モード内で動く役割である。

- Worker:
  考える、調べる、設計する、検証する、判断する。

- Coder:
  読む、書く、直す、テストする、差分を出す。

# 実装の基本方針

以下の方針を必ず含めてください。

- v0.1 では、実際のコード変更を自動実行しない。
- v0.1 では、依頼文を受け取り、モード判定、理由生成、Worker / Coder プロンプト生成、ガードレール判定、作業ログ雛形生成、記憶候補生成までを行う。
- official DB への直接writeはしない。
- user:<uid> への直接upsertはしない。
- memory の confirmed / pinned への直接昇格はしない。
- Source Registry への無審査追加はしない。
- 破壊的操作は実行せず、検出・警告・停止対象にする。
- 最初はCLIで手動確認できる構成にする。
- LangGraphや本物の自律Coder接続は v0.2 以降に回す。
- まず「コーディングAIの運用判断エンジン」として作る。

## 7. Mode Selector 仕様

`mode_selector.py` の仕様を書いてください。

役割:
依頼文や作業対象から Safe Build Mode / Tool Build Mode を判定する。

入力:
- task_text
- optional repo_context
- optional target_files
- optional user_hint

出力:
- ModeDecision

判定ルール:

Safe Build Mode にする例:
- 既存コードを変更する
- 既存DBに触る
- 既存運用に影響する
- 既存環境を変更する
- 本番系設定に触る
- 記憶・Source Registry・validatorに関係する
- 失敗時の影響範囲が大きい

Tool Build Mode にする例:
- 新しい小物ツールを作る
- 既存本体にまだ接続しない
- tools / experiments / sandbox 配下で作れる
- 入出力が明確
- 単体で動作確認できる

判定に迷った場合:
- Safe Build Mode を返す
- requires_human_review = true にする

代表的な入力例と出力例を入れてください。

## 8. Guardrails Checker 仕様

`guardrails.py` の仕様を書いてください。

役割:
依頼内容、計画、想定コマンド、想定変更先を見て、禁止・許可・要確認を判定する。

入力:
- mode
- task_text
- planned_files
- planned_commands
- planned_outputs

出力:
- GuardrailResult

検出すべき禁止事項:
- official DB への直接write
- user:<uid> への直接upsert
- memory の直接確定
- Source Registry への無審査追加
- 検索結果の自動大量投入
- 共有環境の無断変更
- venv / CUDA / モデル配置の無断変更
- 既存ファイルの大規模削除
- 不要なリネームや移動
- Move-Item や rm -rf のような破壊的操作
- テストなしの完了報告
- 3回失敗後の自動継続

要確認にすべきもの:
- DB migration
- config変更
- 既存API変更
- 既存ログ削除
- 依存関係更新
- 環境変数変更
- モデル配置変更

許可されるもの:
- staging/ への出力
- reports/ への出力
- logs/ への出力
- /home/nyukimi/RenCrow/RenCrow_Tools 配下の新規作成
- experiments/ 配下の新規作成
- sandbox/ 配下の新規作成
- memory candidate の作成
- validatorテスト

## 9. Prompt Builder 仕様

`prompt_builder.py` の仕様を書いてください。

役割:
ModeDecision と GuardrailResult をもとに、Worker / Coder 用プロンプトを生成する。

入力:
- task_text
- mode_decision
- guardrail_result
- optional recall_context
- optional repo_context

出力:
- PromptBundle

テンプレート:
- templates/coding_agent/safe_worker.md
- templates/coding_agent/safe_coder.md
- templates/coding_agent/tool_worker.md
- templates/coding_agent/tool_coder.md

Safe Build Mode の Worker Prompt に含めるもの:
- 目的
- 既存仕様を読む
- 影響範囲を出す
- 既存ルール・記憶・過去失敗例を確認する
- テスト観点を作る
- Coder に渡す小さい作業単位を作る

Safe Build Mode の Coder Prompt に含めるもの:
- 関連ファイルを読む
- 指定範囲だけ変更する
- 差分を小さく保つ
- 既存挙動を壊さない
- テストを実行する
- ログを残す
- memory candidate を出す

Tool Build Mode の Worker Prompt に含めるもの:
- 要望整理
- 最小仕様
- 入力と出力
- 保存先ディレクトリ
- README構成
- 将来のRenCrow連携案

Tool Build Mode の Coder Prompt に含めるもの:
- 新規ディレクトリで作る
- READMEを書く
- サンプル入力と実行例を付ける
- テストを書く
- 既存RenCrow本体に直接接続しない
