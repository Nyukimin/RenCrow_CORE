# キャラクターレイヤー表示仕様

## 概要

Mioキャラクターに、ベース画像の上に各顔パーツを重ね合わせるレイヤー構造の表示システムを実装しました。
これにより、「笑ったまま『あいうえお』を言う」といった複合的な表情表現が可能になります。

## 実装内容

### 背景

従来の単一画像切り替え方式では、表情全体を1つの画像として切り替えていたため、
「happy（嬉しい顔）のまま口パク」といった複合表現ができませんでした。

レイヤー方式では、以下のパーツを独立して管理・表示します：

1. **ベース画像** (body_base)
2. **眉毛** (左右2つ)
3. **目** (左右2つ)
4. **口** (1つ)

### パーツ構成

#### Mioの表情パーツ

**眉毛**: 14種類 (eyebrow_01 ~ eyebrow_14)
- 左右ペアで使用

**目**: 14種類 (eyes_01 ~ eyes_14)
- 左右ペアで使用
- 通常の目、細め、ウィンク、驚き、閉じた目など

**口**: 9種類 (mouth_01 ~ mouth_09)
- mouth_01: 閉じた口
- mouth_02: 微笑み
- mouth_03: 笑顔
- mouth_04: 小さく開いた口
- mouth_05: 中くらいの口
- mouth_06: 大きく開いた口（話している）
- mouth_07: 「お」の形
- mouth_08: 一文字口
- mouth_09: への字口

#### 表情の定義

generated/manifest.jsonで、各表情がパーツの組み合わせとして定義されています：

```json
{
  "happy": [
    "eyebrow_13", "eyebrow_14",  // 嬉しそうな眉
    "eyes_11", "eyes_12",         // 細めた目
    "mouth_03"                    // 笑顔の口
  ],
  "normal": [
    "eyebrow_01", "eyebrow_02",  // 通常の眉
    "eyes_01", "eyes_02",         // 通常の目
    "mouth_01"                    // 閉じた口
  ]
}
```

### バックエンド実装

#### 1. レイヤーマネージャー (`character_layered.go`)

- `LayeredCharacterManifest`: generated/manifest.jsonの構造を表現
- `LayeredCharacterManager`: マニフェストを読み込み、パーツ情報を管理
- `GetCharacterState()`: 表情名からパーツURLの組み合わせを取得
- `SetMouthPart()`: 口パーツだけを変更

#### 2. APIエンドポイント (`character_layered_handler.go`)

**エンドポイント**:

```
GET /viewer/character/layered/state?character_id=mio&expression=happy
```

レスポンス:
```json
{
  "character_id": "mio",
  "expression": "happy",
  "parts": {
    "base": "/viewer/assets/images/mio/parts/generated/base/mio_body_base.png",
    "eyebrow_left": "/viewer/assets/.../eyebrow_13.png",
    "eyebrow_right": "/viewer/assets/.../eyebrow_14.png",
    "eye_left": "/viewer/assets/.../eyes_11.png",
    "eye_right": "/viewer/assets/.../eyes_12.png",
    "mouth": "/viewer/assets/.../mouth_03.png"
  },
  "available_expressions": ["happy", "normal", "smile", ...],
  "available_mouths": ["mouth_01", "mouth_02", ...]
}
```

```
GET /viewer/character/layered/mouth?character_id=mio&expression=happy&mouth_id=mouth_06
```

口パーツだけを変更したStateを返します。

```
GET /viewer/character/layered/manifest?character_id=mio
```

完全なマニフェスト情報を返します。

### フロントエンド実装

#### 1. HTML構造

レイヤー表示用のHTML構造:

```html
<div class="character-layered">
  <img class="character-part character-base" src="" alt="Mio base">
  <img class="character-part character-eyebrow-left" src="" alt="Left eyebrow">
  <img class="character-part character-eyebrow-right" src="" alt="Right eyebrow">
  <img class="character-part character-eye-left" src="" alt="Left eye">
  <img class="character-part character-eye-right" src="" alt="Right eye">
  <img class="character-part character-mouth" src="" alt="Mouth">
</div>
```

#### 2. CSS

各パーツを絶対配置で重ね合わせ:

```css
.character-layered {
  position: relative;
  width: 100%;
  height: 100%;
}

.character-part {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  object-fit: contain;
}

.character-base { z-index: 1; }
.character-eyebrow-left, .character-eyebrow-right { z-index: 2; }
.character-eye-left, .character-eye-right { z-index: 3; }
.character-mouth { z-index: 4; }
```

#### 3. JavaScript

主要な関数:

```javascript
// 表情全体を切り替え
loadLayeredCharacterState(characterId, expressionName)

// 口パーツだけを変更
setLayeredMouth(characterId, mouthId)

// レイヤーパーツを更新
updateLayeredCharacterParts(characterId)
```

## 使用例

### 1. 表情の切り替え

```javascript
// Mioをhappyにする
setCharacterExpression('mio', 'happy');
// → 眉毛、目、口がすべてhappy表情のパーツに切り替わる
```

### 2. 表情を保ちながら口パクする

```javascript
// happy表情のままで口パク
// 1. happy表情にする
await loadLayeredCharacterState('mio', 'happy');

// 2. 口を開く（話している）
await setLayeredMouth('mio', 'mouth_06');

// 3. 口を閉じる
await setLayeredMouth('mio', 'mouth_01');
```

このように、happy表情（細めた目、上がった眉毛）を保ちながら、
口だけを切り替えて口パクができます。

### 3. 「あいうえお」口パク

```javascript
// happy表情で「あいうえお」
await loadLayeredCharacterState('mio', 'happy');

// あ
await setLayeredMouth('mio', 'mouth_06');
await sleep(200);

// い
await setLayeredMouth('mio', 'mouth_02');
await sleep(200);

// う
await setLayeredMouth('mio', 'mouth_04');
await sleep(200);

// え
await setLayeredMouth('mio', 'mouth_05');
await sleep(200);

// お
await setLayeredMouth('mio', 'mouth_07');
```

## 動作フロー

### 初期化時

1. `characterStates['mio'].layered = true` を設定
2. `loadLayeredCharacterState('mio', 'normal')` を実行
3. APIから各パーツのURLを取得
4. `updateLayeredCharacterParts()` でDOM要素を更新

### 表情切り替え時

1. エージェントイベント（thinking, happy, errorなど）を受信
2. `setCharacterExpression('mio', 'happy')` を呼び出し
3. `loadLayeredCharacterState()` が実行される
4. APIから新しい表情のパーツを取得
5. すべてのパーツ画像が更新される

### 口パク時

1. TTS再生開始
2. `setLipSyncSpeaking('mio', true)` が呼ばれる
3. レイヤー表示を使用する場合、`setLayeredMouth('mio', 'mouth_06')` を実行
4. 口パーツだけが更新される（眉毛と目はそのまま）
5. TTS再生終了
6. `setLipSyncSpeaking('mio', false)` が呼ばれる
7. 現在の表情の通常の口パーツに戻る

## 複合表現の例

### 例1: 笑いながら話す

```javascript
// 1. happy表情にする（細めた目、上がった眉毛、笑顔の口）
await loadLayeredCharacterState('mio', 'happy');

// 2. TTS再生時、口だけをmouth_06に変更
await setLayeredMouth('mio', 'mouth_06');

// → 細めた目と上がった眉毛はそのまま、口だけが開いた状態
```

### 例2: 困った顔で話す

```javascript
// 1. trouble表情にする（困った眉毛、困った目、への字口）
await loadLayeredCharacterState('mio', 'trouble');

// 2. TTS再生時、口だけをmouth_06に変更
await setLayeredMouth('mio', 'mouth_06');

// → 困った表情を保ちながら話す
```

### 例3: 考えながら話す

```javascript
// 1. think表情にする
await loadLayeredCharacterState('mio', 'think');

// 2. 口パク
await setLayeredMouth('mio', 'mouth_06');
```

## Shiroキャラクターの対応

現在、Shiroは従来の単一画像方式を使用しています。
将来的にShiro用のレイヤーパーツを作成すれば、同様のレイヤー表示が可能です。

Shiro対応の手順：
1. Shiro用のパーツ画像を作成（generated/ディレクトリ構造）
2. generated/manifest.jsonを作成
3. `characterStates['shiro'].layered = true` に設定
4. HTMLにShiro用のレイヤー構造を追加

## パフォーマンス

### メリット

- 画像の差分更新が可能（口だけ変更など）
- 表情のバリエーションが増える（パーツの組み合わせ）
- メモリ効率が良い（全表情×口パターンの画像を用意する必要がない）

### 注意点

- 初回読み込み時に複数の画像を読み込む必要がある
- ブラウザのキャッシュを活用することで、2回目以降は高速
