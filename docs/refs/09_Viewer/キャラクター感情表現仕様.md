# キャラクター感情表現仕様

## 概要

MioとShiroのキャラクター表示に、感情に応じた表情切り替え機能を実装しました。
各キャラクターのmanifest.jsonに定義された表情パーツと状態マッピングを使用して、
エージェントの状態やTTS再生状態に応じて自動的に表情を切り替えます。

## 実装内容

### バックエンド

#### 1. キャラクターマニフェスト管理 (`internal/adapter/viewer/character_manifest.go`)

- `CharacterManifest`: manifest.jsonの構造を表現する構造体
- `CharacterManager`: キャラクターのマニフェストを読み込み、管理するシングルトン
- `GetExpressionURL()`: 状態名から表情画像URLを取得
- `GetBodyURL()`: 状態名からボディポーズ画像URLを取得
- `GetAvailableStates()`: 利用可能な状態一覧を取得

#### 2. APIエンドポイント (`internal/adapter/viewer/character_state_handler.go`)

**エンドポイント**:
- `GET /viewer/character/state?character_id=<id>&state=<state>`
  - キャラクターの状態情報を取得
  - レスポンス: `CharacterStateResponse` (expression_url, body_url, talk_open_url, talk_closed_url, available_states)

- `GET /viewer/character/manifest?character_id=<id>`
  - キャラクターのmanifest.json全体を取得

#### 3. ルーティング (`cmd/rencrow/routes.go`)

- `/viewer/character/state` と `/viewer/character/manifest` を登録

### フロントエンド

#### 1. キャラクター状態管理 (`internal/adapter/viewer/assets/js/viewer.js`)

**追加された機能**:

- `characterStates`: MioとShiroの現在の状態を管理するオブジェクト
- `loadCharacterState(characterId, stateName)`: APIから状態を取得して適用
- `setCharacterExpression(characterId, stateName)`: 表情を設定
- `updateCharacterImages(characterId, expressionURL, talkOpenURL)`: すべてのキャラクター画像要素を更新

**自動表情切り替え**:

1. **エージェント状態イベント**:
   - `agent.thinking` → thinking状態
   - `agent.response` (成功) → happy/idle状態
   - `agent.response` (エラー) → error状態

2. **TTS再生**:
   - `startLipSyncInternal()` → speaking状態
   - `stopLipSyncInternal()` → idle状態

3. **ページロード時**:
   - MioとShiroを初期状態(idle)で初期化

## 使用例

### APIから状態を取得

```bash
# Mioのthinking状態を取得
curl "http://localhost:18790/viewer/character/state?character_id=mio&state=thinking"

# レスポンス例
{
  "character_id": "mio",
  "state": "thinking",
  "expression_url": "/viewer/assets/images/mio/parts/expressions/mio_think.png",
  "body_url": "/viewer/assets/images/mio/parts/body/mio_body_left.png",
  "talk_open_url": "/viewer/assets/images/mio/parts/expressions/mio_talk_open.png",
  "talk_closed_url": "/viewer/assets/images/mio/parts/expressions/mio_think.png",
  "available_states": ["idle", "thinking", "speaking", "happy", ...]
}
```

### JavaScriptから表情を切り替え

```javascript
// Mioをhappy状態に切り替え
setCharacterExpression('mio', 'happy');

// Shiroをthinking状態に切り替え
setCharacterExpression('shiro', 'thinking');
```

## 動作フロー

1. **ページロード時**:
   - `loadCharacterState('mio', 'idle')`
   - `loadCharacterState('shiro', 'idle')`

2. **エージェントイベント受信時**:
   - イベントタイプと内容を解析
   - 適切な状態を判定
   - `setCharacterExpression()` で表情を更新

3. **TTS再生時**:
   - `startLipSyncInternal()` → speaking状態
   - 音声再生終了 → idle状態

4. **APIリクエスト**:
   - `/viewer/character/state` から状態情報を取得
   - `updateCharacterImages()` ですべての画像要素を更新

## 更新対象の画像要素

1. **Lipsync Actor** (`#lipSyncMio`, `#lipSyncShiro`)
   - 画面上部の口パク用キャラクター画像
   - data-openとdata-closedを更新

2. **Chat Character Avatar** (`.chat-character-avatar`)
   - Chatパネルのキャラクター画像（Mioのみ）

3. **Idle Character Avatar** (`.idle-character-avatar.mio`, `.idle-character-avatar.shiro`)
   - IdleChatパネルのキャラクター画像

## 今後の拡張可能性

1. **新しい表情の追加**:
   - manifest.jsonのexpressionsに新しいパーツを追加
   - state_mapに新しい状態マッピングを定義

2. **ボディポーズの活用**:
   - 現在は表情のみ切り替えているが、ボディポーズも切り替え可能

3. **アニメーション**:
   - CSS transitionやアニメーションを追加して滑らかな切り替え

4. **カスタム状態**:
   - ユーザー定義の状態や表情パターンの追加
