# 分割再設計候補

## 目的

この文書は、仕様変更時に実装箇所が 1 対 1 で説明しにくくなった箇所を記録する。

Phase34 時点では、追加分割すべき未分化 production module は残っていない。ただし、行数が残る単一責務ファイルや、今後の仕様追加で複数変更理由を持つ可能性のある領域はある。

## 現時点の判定

Phase34 の監査では、Phase28 から Phase33 で残存していた大型かつ未分化な production module は責務別に分割済みである。

現時点で追加分割すべき対象はない。

## 監視対象

次のファイルは、現時点では単一責務として説明できるため追加分割不要だが、今後の仕様追加で複数責務を持ち始めた場合は再設計候補にする。

| 対象 | 現在の判定 | 再設計が必要になる条件 |
| --- | --- | --- |
| `internal/adapter/config/config_types.go` | config type 定義 | type 定義以外の validation / runtime logic が増える |
| `internal/infrastructure/persistence/conversation/vectordb_kb.go` | VectorDB KB API | thread memory、archive、Source Registry まで混ざる |
| `internal/infrastructure/transport/ssh.go` | SSH transport | transport contract と agent protocol が混ざる |
| `internal/application/idlechat/forecast_trend_sources.go` | forecast trend source fetch / rank | session runner、topic stock、Viewer 表示が混ざる |
| `internal/infrastructure/tts/sbv2_provider.go` | SBV2 provider | bridge、audio router、Viewer 表示処理が混ざる |
| `internal/adapter/config/config_defaults.go` | default 設定 | validation、live runtime resolution が混ざる |
| `internal/domain/conversation/recall_pack.go` | RecallPack domain contract | provider call、DB query、Viewer 表示が混ざる |
| `internal/infrastructure/llm/providers/claude/provider.go` | Claude provider | stream、thinking bridge、model ready などが増え、provider 本体が肥大化する |
| `cmd/rencrow/stt_runtime_websocket.go` | STT websocket runtime | provider 本体、Viewer 表示、IdleChat 入力が混ざる |
| `internal/domain/agent/coder.go` | Coder proposal generation | Worker execution や破壊的操作が混ざる |

## 再設計が必要な兆候

- 1 つの仕様変更で 3 つ以上の主担当パッケージを同時に変更する。
- `cmd/rencrow` に業務判断が戻る。
- `helper`、`manager`、`util` が新しい集中点になる。
- Viewer 表示、音声、口パク、ログが同じ関数や DTO に混ざる。
- fallback を成功として隠す。
- Source Registry の observed / candidate / validated / promoted が飛ばされる。
- repo example と live runtime config の意味が混ざる。

## 記録ルール

再設計候補を追加するときは、次を書く。

- 仕様変更の目的
- 現在の主担当ファイル
- 1 対 1 に説明できない理由
- 分割候補
- 守る契約
- 先に必要な検証

未確定のままの文言は残さない。現時点で判断できる内容だけを記録する。
