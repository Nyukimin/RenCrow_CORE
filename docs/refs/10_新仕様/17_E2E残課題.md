## 5. 判定ルール

- skip は成功ではなく、環境未準備または未確認として扱う。
- fallback は成功ではなく、期待 route 不成立として扱う。
- health ok だけで user flow 成立扱いにしない。
- handler / stub / unit test の成功だけで実 API E2E 完了扱いにしない。
- 実ブラウザ UI は、DOM 存在ではなく visible state と実操作の成立で判定する。
- docs の状態表を更新する場合は、E2E の確認範囲と skip 理由を併記する。
