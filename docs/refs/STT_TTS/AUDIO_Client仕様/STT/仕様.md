# RenCrow_STT 仕様（AUDIO_Client向け）

本書は、Chat から RenCrow_STT を操作するための外部契約仕様です。  
TTS と同一ホストで運用する前提とし、STT は `:8090` を利用します。

---

## 1. 目的

- Chat から STT を呼び出す経路を `WS /stt` に統一する
- 音声入力から確定文字列（`final`）までの契約を固定化する
- エラー時の再接続/通知ルールを明確化する

---

## 2. スコープ

### 対象

- Chat -> RenCrow_STT の HTTP/WS 契約
- Client 送信制御（`config` / `final_pending`）
- Server 送信イベント（`session_info` / `final` / `error` ほか）

### 非対象

- STT エンジン内部（whisper.cpp / faster-whisper）実装詳細
- LLM 応答生成
- TTS 音声合成処理

---

## 3. エンドポイント契約

### 3.1 HTTP ヘルスチェック

- `GET /health`
  - 用途: プロセス生存確認
  - 成功例: `{ "ok": true }`
- `GET /ready`
  - 用途: 推論エンジン ready 確認
  - 成功例:

```json
{
  "ready": true,
  "engine": "whispercpp",
  "engine_detail": { "detail": "status 200" }
}
```

注記: `ready=false` でも HTTP ステータスは 200 を返す。

### 3.2 WebSocket 接続（主経路）

- 主経路: `WS /stt`
- 互換経路: `WS /ws`, `WS /stt-ws`（非推奨）

接続URL例（TTS と同一IP）:

- 開発: `ws://192.168.1.36:8090/stt`
- LAN/本番: `wss://192.168.1.36:8090/stt`（TLS 終端あり）

---

## 4. メッセージ契約

### 4.1 Client -> STT

`config`:

```json
{ "type": "config", "mimeType": "audio/wav" }
```

`vad`（現状 no-op）:

```json
{ "type": "vad", "speaking": true }
```

`final_pending`:

```json
{ "type": "final_pending" }
```

音声バイナリ:

- 推奨: WAV PCM16 16kHz mono
- `config` 送信後にバイナリチャンクを順次送信

### 4.2 STT -> Client

- `session_info`: 接続直後に送信
- `speech_start`: 発話開始検出
- `draft`: 中間結果（`RENCROW_STT_DRAFT_ENABLED=true` 時のみ）
- `final`: 確定文字列
- `status`: 状態通知
- `error`: 契約違反/推論失敗通知

`final` 例:

```json
{ "type": "final", "text": "こんにちは" }
```

`error` 例:

```json
{ "type": "error", "code": "PROVIDER_UNAVAILABLE", "text": "backend unavailable" }
```

---

## 5. Chat 側の利用ルール

- Chat は確定結果として `final` のみを利用する
- `reply_reset` / `reply_delta` は RenCrow_STT から送出されない前提で実装する
- `final` を受信したら既存 Router/Worker へ user 発話として投入する
- `error.code` で再接続/通知を分岐する（文言一致で分岐しない）

---

## 6. エラーコード契約

代表コード:

- `PROVIDER_UNAVAILABLE`
- `PROVIDER_TIMEOUT`
- `PROVIDER_HTTP_ERROR`
- `PROVIDER_EXCEPTION`
- `AUDIO_TOO_SHORT`
- `INVALID_PAYLOAD`

処理方針:

- `PROVIDER_*`: バックオフ再接続
- `INVALID_PAYLOAD`: 音声形式を見直して再送
- `AUDIO_TOO_SHORT`: 収録長を延ばして再試行

---

## 7. 環境変数（推奨）

```env
RENCROW_STT_URL=ws://192.168.1.36:8090/stt
RENCROW_STT_CONNECT_TIMEOUT_MS=5000
RENCROW_STT_RECONNECT_BACKOFF_MS=2000,5000,10000
```

IP は TTS と同一ホストを指定する。

---

## 8. 受け入れチェックリスト

1. `GET /health` が `{"ok":true}`
2. `GET /ready` が `{"ready":true,...}`（または未ready理由を返却）
3. `WS /stt` 接続直後に `session_info` 受信
4. 音声送信で `final` を受信
5. `PROVIDER_UNAVAILABLE` 時に再接続バックオフが動作
6. `reply_*` 非依存で会話継続が可能
