# RenCrow プロンプト配置運用ガイド

**作成日**: 2026-05-09  
**対象**: RenCrow の SystemPrompt / persona / SOUL / コーディングルールを運用中に編集したい場合

---

## 1. 基本方針

RenCrow のプロンプトは、次の役割分担で管理する。

```text
repo 側 prompts/
= フォールバック、初期値、配布用

~/.rencrow/workspace/
= 運用中に編集する正本、RenCrow が書き換えてよい領域
```

ローンチ後も RenCrow 自身がプロンプトを書き換える前提なら、運用正本は `~/.rencrow/workspace/` に集約する。
各エージェントのキャラクター bundle は `~/.rencrow/workspace/prompts/characters/<agent>/` を正本にする。
repo 側 `prompts/characters/` は参照しない。

---

## 2. 推奨 config

`~/.rencrow/config.yaml` では、repo 側の初期プロンプトと、運用中に編集する workspace を明確に分ける。

```yaml
prompts_dir: /home/nyukimi/RenCrow_CORE/prompts
workspace_dir: /home/nyukimi/.rencrow/workspace
```

`workspace_dir: ./workspace` のような相対パスにすると、systemd の `WorkingDirectory` に依存する。運用環境では絶対パスを推奨する。

---

各エージェントの SystemPrompt / policy / routing / knowledge は、次の Prompt Bundle 形式で読み込まれる。

```text
~/.rencrow/workspace/prompts/characters/mio/
~/.rencrow/workspace/prompts/characters/shiro/
~/.rencrow/workspace/prompts/characters/aka/
~/.rencrow/workspace/prompts/characters/ao/

```

各ディレクトリは `manifest.txt` を持ち、読み込み順を固定する。

```text
manifest.txt
00_system.md
10_policy.md
20_routing.md
30_knowledge.md
```

repo 側 `prompts/characters/` は参照しない。
workspace 側の Mio bundle を運用正本として扱う。

プロンプトファイルは起動時に読み込まれるため、変更後はサービス再起動が必要。

## 4. SystemPrompt と persona の違い

SystemPrompt は、その LLM 呼び出し全体を制御する実行指示である。

```text
SystemPrompt
= 何をどう処理するか
= 出力形式、禁止事項、ツール利用、実装ルール、分類ルール
```

persona は、キャラ別の人格・振る舞い・口調ルールである。

```text
persona/*.md
= 誰として振る舞うか
= 性格、口調、価値観、判断傾向、得意分野
```

Coder / Worker では、persona は SystemPrompt の前に前置される。

```text
persona/ao.md

+

coder.md
```

そのため、強い実装ルールや出力形式は `coder.md` / `worker.md` に置き、`persona/*.md` はキャラ性と判断傾向に留める。

## 5. 用途別の置き場所

### 5.1 Chat / Mio

```text
~/.rencrow/workspace/prompts/characters/mio/
```

Mio の通常会話 SystemPrompt。会話の基本人格・応答方針として最も直接効く。

```text
~/.rencrow/workspace/SOUL.md
~/.rencrow/workspace/IDENTITY.md
~/.rencrow/workspace/USER.md
~/.rencrow/workspace/PrimerMessage.md
```

Chat 専用の長期方針、自己定義、ユーザー嗜好、初期文脈。現行の Context Builder で参照される。

### 5.2 Worker / Shiro

```text
~/.rencrow/workspace/prompts/characters/shiro/
```

Worker の実行 SystemPrompt。コマンド実行、パッチ適用、検証、失敗時の立て直しルールを置く。

Shiro の人格・口調・実行担当としての振る舞いも同じ bundle に置く。

### 5.3 Coder 共通

```text
~/.rencrow/workspace/coder.md
```

全 Coder 共通の SystemPrompt。Plan / Patch / Risk 形式、検証義務、禁止事項、プロジェクト固有のコーディングルールを置く。

`rules/PROJECT_AGENT.md` や `rules/common/*.md` は正本ドキュメントだが、現行実装では Coder が自動では読まない。実行時に確実に効かせたいルールは `workspace/coder.md` に要約して置く。

### 5.4 Coder 個別

```text
~/.rencrow/workspace/prompts/characters/aka/  # coder1
~/.rencrow/workspace/prompts/characters/ao/   # coder2
~/.rencrow/workspace/prompts/characters/gin/  # coder3
~/.rencrow/workspace/prompts/characters/kin/  # coder4
```

各 Coder の人格、得意分野、判断傾向を置く。

### 5.5 IdleChat

```text
~/.rencrow/workspace/idle_chat/mio.md
~/.rencrow/workspace/idle_chat/shiro.md
~/.rencrow/workspace/idle_chat/aka.md
~/.rencrow/workspace/idle_chat/ao.md
~/.rencrow/workspace/idle_chat/gin.md
```

IdleChat 専用の会話人格。通常会話の `mio.md` や Coder persona とは分ける。

### 5.6 Routing / Classifier

```text
~/.rencrow/workspace/classifier.md
```

ルーティング分類器の SystemPrompt。`CHAT` / `OPS` / `CODE` / `CODE2` などの分類方針を置く。

## 7. RenCrow が書き換える場合の安全ルール

RenCrow 自身にプロンプト編集を許可する場合は、書き込み範囲を限定する。

推奨:

```text
書き込み許可:
~/.rencrow/workspace/
~/.rencrow/workspace/prompts/characters/
~/.rencrow/workspace/persona/
~/.rencrow/workspace/idle_chat/
~/.rencrow/workspace/prompt_history/

書き込み禁止:
repo 側 prompts/
rules/
docs/
```

編集時は、変更前の内容を `prompt_history/` に退避する。

```text
~/.rencrow/workspace/prompt_history/
  20260509_153000_coder.md
  20260509_153500_persona_ao.md
```

---

## 8. 結論

RenCrow 自身が書き換える運用上の正本は `~/.rencrow/workspace/` に集約する。

repo 側の `prompts/` は legacy prompt のフォールバック、初期値、配布用として残す。キャラクター bundle は `~/.rencrow/workspace/prompts/characters/<agent>/` を正本にする。

SystemPrompt は処理ルール、persona は人格・口調・判断傾向として分ける。

コーディング時に確実に効かせたいルールは `~/.rencrow/workspace/coder.md` に置く。
