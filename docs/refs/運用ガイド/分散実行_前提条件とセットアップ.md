# 分散実行 - 前提条件とセットアップガイド

**作成日**: 2026-03-26
**対象**: Worker と別マシンで Chat/Coder を実行する場合の前提条件

---

## 1. 概要

RenCrow (RenCrow_CORE) の分散実行モードでは、Worker（Linux）と Coder（Windows等）を別マシンで実行できます。

**構成例**:
```
Linux (Worker)
  ├─ Mio (Chat)
  ├─ Shiro (Worker)
  ├─ Coder1 (AO - DeepSeek) ← ローカル
  ├─ Coder2 (Aka - OpenAI)    ← ローカル
  └─ SSH Transport
       ↓
Windows (Remote)
  ├─ Coder3 (Kin - Claude)   ← SSH経由
  └─ Coder4 (Gin - Gemini)   ← SSH経由
```

---

## 2. 前提条件（必須）

### 2.1 SSH接続が確立されていること

**必須要件**:
- Worker マシンから Remote マシンへ SSH 接続可能
- SSH 鍵認証が設定済み（パスワード認証は非推奨）
- Remote マシンで SSH サーバーが起動中

---

### 2.2 ネットワーク疎通

**チェック項目**:
- Worker → Remote への TCP/22 (SSH) 通信が許可されている
- ファイアウォール設定で SSH ポートが開放されている
- Tailscale/VPN 等を使用する場合、接続が安定していること

---

### 2.3 Remote マシンの準備

**必要なファイル**:
- `rencrow-agent` バイナリ（Windows: `.exe`）
- SSH サーバー（Windows: OpenSSH Server）

**不要なもの**:
- `config.yaml` （Worker から設定を受信するため不要）
- API キー（Worker の環境変数を使用）

---

## 3. SSH セットアップ手順

### 3.1 Linux (Worker) 側

#### 1. SSH 鍵ペア生成（未作成の場合）

```bash
# ED25519 鍵を生成（推奨）
ssh-keygen -t ed25519 -C "rencrow-worker"

# 保存先: ~/.ssh/id_ed25519
# パスフレーズは任意（推奨: 設定する）
```

#### 2. 公開鍵を Remote へコピー

```bash
# Remote マシンへ公開鍵を転送
ssh-copy-id -i ~/.ssh/id_ed25519.pub nyukimin@bmax2nd-1

# 手動の場合
cat ~/.ssh/id_ed25519.pub
# → Remote の ~/.ssh/authorized_keys に追記
```

#### 3. SSH 接続テスト

```bash
# パスワード入力なしで接続できることを確認
ssh -i ~/.ssh/id_ed25519 nyukimin@bmax2nd-1

# 成功したら exit
```

#### 4. known_hosts 登録

```bash
# 初回接続時に fingerprint を確認して yes
ssh nyukimin@bmax2nd-1
# The authenticity of host '...' can't be established.
# ED25519 key fingerprint is SHA256:...
# Are you sure you want to continue connecting (yes/no)? yes

# または ssh-keyscan で事前登録
ssh-keyscan -H bmax2nd-1 >> ~/.ssh/known_hosts
```

---

### 3.2 Windows (Remote) 側

#### 1. OpenSSH Server インストール

```powershell
# Windows 10/11 の場合
# 設定 → アプリ → オプション機能 → OpenSSH Server 追加

# PowerShell で確認
Get-WindowsCapability -Online | Where-Object Name -like 'OpenSSH.Server*'

# インストール
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
```

#### 2. SSH サービス起動

```powershell
# サービス開始
Start-Service sshd

# 自動起動設定
Set-Service -Name sshd -StartupType 'Automatic'

# 確認
Get-Service sshd
```

#### 3. ファイアウォール設定

```powershell
# SSH ポート (22) を許可
New-NetFirewallRule -Name sshd -DisplayName 'OpenSSH Server (sshd)' `
  -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22
```

#### 4. 公開鍵認証設定

```powershell
# ~/.ssh/authorized_keys に Worker の公開鍵を追加
# (ssh-copy-id を使った場合は自動設定済み)

# 手動の場合
# C:\Users\nyukimin\.ssh\authorized_keys に Worker の公開鍵を貼り付け

# パーミッション設定（重要！）
icacls C:\Users\nyukimin\.ssh\authorized_keys /inheritance:r
icacls C:\Users\nyukimin\.ssh\authorized_keys /grant:r "nyukimin:F"
icacls C:\Users\nyukimin\.ssh\authorized_keys /grant SYSTEM:F
```

#### 5. rencrow-agent バイナリ配置

```powershell
# ビルド（Linux側で実行）
GOOS=windows GOARCH=amd64 go build -o rencrow-agent.exe ./cmd/rencrow-agent

# Windows へ転送
scp rencrow-agent.exe nyukimin@bmax2nd-1:C:/rencrow/

# 配置確認
Test-Path C:\rencrow\rencrow-agent.exe
```

## 4. 接続確認

### 4.1 手動テスト

```bash
# Linux (Worker) から実行

# 1. SSH 接続確認
ssh nyukimin@bmax2nd-1 "echo 'SSH OK'"
# 出力: SSH OK

# 2. rencrow-agent 起動確認
ssh nyukimin@bmax2nd-1 "C:/rencrow/rencrow-agent.exe --version"
# 出力: rencrow-agent version x.x.x

# 3. JSON 通信テスト
echo '{"from":"test","to":"coder3","session_id":"test","job_id":"test","message":"ping","timestamp":"2026-03-26T00:00:00Z"}' | \
  ssh nyukimin@bmax2nd-1 "C:/rencrow/rencrow-agent.exe --standalone"
```

### 4.2 自動テスト

```bash
# config.yaml の distributed セクションを確認
grep -A 20 "distributed:" ~/.rencrow/config.yaml

# rencrow 起動（分散モード有効）
./rencrow --config ~/.rencrow/config.yaml

# ログで SSH 接続確認
tail -f ~/.rencrow/logs/rencrow.log | grep SSH
```

## 5. トラブルシューティング

### 5.1 SSH 接続エラー

**症状**: `ssh: connect to host bmax2nd-1 port 22: Connection refused`

**原因と対処**:
1. Remote の SSH サーバーが起動していない
   ```powershell
   # Windows で確認
   Get-Service sshd
   Start-Service sshd
   ```

2. ファイアウォールが SSH をブロック
   ```powershell
   # Windows ファイアウォール確認
   Get-NetFirewallRule -Name sshd
   ```

3. ネットワーク疎通不可
   ```bash
   # ping 確認
   ping bmax2nd-1

   # Tailscale 確認
   tailscale status
   ```

---

### 5.2 認証エラー

**症状**: `Permission denied (publickey)`

**原因と対処**:
1. 公開鍵が正しく登録されていない
   ```bash
   # 公開鍵を再送信
   ssh-copy-id -i ~/.ssh/id_ed25519.pub nyukimin@bmax2nd-1
   ```

2. Windows の authorized_keys パーミッション問題
   ```powershell
   # パーミッション再設定
   icacls C:\Users\nyukimin\.ssh\authorized_keys /inheritance:r
   icacls C:\Users\nyukimin\.ssh\authorized_keys /grant:r "nyukimin:F"
   ```

3. SSH 鍵のパスフレーズ
   ```bash
   # ssh-agent にキーを追加
   eval $(ssh-agent)
   ssh-add ~/.ssh/id_ed25519
   ```

---

### 5.3 rencrow-agent 起動失敗

**症状**: `rencrow-agent: command not found`

**原因と対処**:
1. パスが間違っている
   ```bash
   # config.yaml で正しいパスを指定
   remote_agent_path: "C:/rencrow/rencrow-agent.exe"
   ```

2. 実行権限がない（Linux の場合）
   ```bash
   chmod +x /path/to/rencrow-agent
   ```

3. バイナリが存在しない
   ```bash
   # 再ビルド・再転送
   GOOS=windows GOARCH=amd64 go build -o rencrow-agent.exe ./cmd/rencrow-agent
   scp rencrow-agent.exe nyukimin@bmax2nd-1:C:/rencrow/
   ```

---

### 5.4 JSON 通信エラー

**症状**: `invalid character '\x00' looking for beginning of value`

**原因と対処**:
1. rencrow-agent のバージョン不一致
   ```bash
   # Worker と Remote で同じバージョンを使用
   ./rencrow --version
   ssh nyukimin@bmax2nd-1 "C:/rencrow/rencrow-agent.exe --version"
   ```

2. 標準入出力が汚染されている
   ```bash
   # SSH 設定で Banner を無効化
   # Remote の /etc/ssh/sshd_config
   Banner none
   ```

## 6. セキュリティ考慮事項

### 6.1 SSH 鍵の保護

```bash
# 秘密鍵のパーミッション（必須）
chmod 600 ~/.ssh/id_ed25519

# パスフレーズ設定（推奨）
ssh-keygen -p -f ~/.ssh/id_ed25519
```

### 6.2 known_hosts の検証

```yaml
# config.yaml で strict モード有効化（本番推奨）
distributed:
  transports:
    coder3:
      strict_host_key: true  # known_hosts 必須
```

### 6.3 API キーの扱い

```bash
# Worker の環境変数のみに API キーを配置
# ~/.rencrow/.env
export ANTHROPIC_API_KEY="sk-ant-..."

# パーミッション設定
chmod 600 ~/.rencrow/.env

# Remote には API キーを置かない
# → Worker から SSH 経由で送信される
```
