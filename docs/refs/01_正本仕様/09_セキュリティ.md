## 1. API キー管理

### 1.1 取得方法

API キーは**環境変数から取得**する。設定ファイルへの平文保存は禁止。

| キー | 環境変数 |
|------|---------|
| Anthropic Claude | `ANTHROPIC_API_KEY` |
| OpenAI | `OPENAI_API_KEY` |
| DeepSeek | `DEEPSEEK_API_KEY` |

**推奨設定**:

```bash
# ~/.rencrow/.env (chmod 600)
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
DEEPSEEK_API_KEY=...
```

systemd からの読み込み:

```ini
[Service]
EnvironmentFile=/home/nyukimi/.rencrow/.env
```

### 1.2 config.yaml での参照

`${ENV_VAR}` 展開（`os.ExpandEnv`）で間接参照可能。ただし平文値の直書きは禁止。

```yaml
providers:
  claude:
    api_key: "${ANTHROPIC_API_KEY}"  # 環境変数参照（直書き禁止）
```

### 1.3 ログマスキング

ログ出力時に API キーを自動マスク（`pkg/logger/masking.go`）:

```
ANTHROPIC_API_KEY=sk-ant-1234... → ANTHROPIC_API_KEY=sk-ant-****
```

## 2. クラウド LLM 利用制限

### 2.1 CODE 以外クラウド禁止ルール

**ルール**: `CHAT`, `PLAN`, `ANALYZE`, `OPS`, `RESEARCH` ルートはクラウド LLM への送信禁止

**理由**: ユーザーの日常会話や運用コマンドが外部に送信されることを防ぐ

**実装**: `local_only` フラグによるチェック

### 2.2 `/local` / `/cloud` コマンド

| コマンド | 効果 |
|---------|------|
| `/local` | クラウド利用禁止フラグを立てる（`local_only=true`） |
| `/cloud` | `/local` を解除する |

`local_only=true` 中に `/code` が来た場合: CODE 実行を拒否し、`/cloud` で解除を案内。

## 3. Worker セーフガード

### 3.1 必須セーフガード一覧

| セーフガード | 説明 | 必須/推奨 |
|------------|------|---------|
| **詳細ログ記録** | 実行前後に plan/patch・結果・変更ファイルをログ | 必須 |
| **保護ファイルパターン** | `.env*`, `*credentials*`, `*.key`, `*.pem` への操作は即エラー | 推奨 |
| **ワークスペース制限** | ファイル操作は `workspace` 設定内のみ許可 | 必須 |
| **Git auto-commit** | 実行前後に自動コミット（`git revert` で即巻き戻し可能） | 推奨 |
| **実行前サマリ表示** | 変更内容の概要を実行前にログ出力 | 推奨 |
| **ドライモード** | `RENCROW_DRY_RUN=true` で実行内容のみログ出力 | オプション |

### 3.2 保護ファイル設定

```yaml
worker:
  protected_patterns:
    - ".env*"
    - "*credentials*"
    - "*.key"
    - "*.pem"
  action_on_protected_file: "error"  # "error", "skip", "log"
```

### 3.3 タイムアウト

- シェルコマンド: 5分（300秒）
- Git 操作: 30秒

## 4. 入力バリデーション（ツール系）

### 4.1 パストラバーサル防止

```go
func validatePath(path string) error {
    if strings.Contains(path, "..") {
        return fmt.Errorf("path traversal detected: %s", path)
    }
    return nil
}
```

### 4.2 制御文字拒否

拒否: `\x00-\x08`, `\x0b-\x0c`, `\x0e-\x1f`
許容: `\x09` (TAB), `\x0a` (LF), `\x0d` (CR)

### 4.3 入力長制限

| フィールド種別 | 最大長 |
|-------------|------|
| ID | 256 |
| タイトル / 名前 | 500 |
| 説明文 | 5,000 |
| 本文 / コンテンツ | 50,000 |
| URL | 2,048 |
| 配列要素数 | 1,000 |

### 4.4 コマンドインジェクション対策

Worker がシェルコマンドを実行する際、適切なエスケープ処理を実施。生成されたコマンドのログ記録は必須。

## 5. ToolRegistry 信頼モデル

（ケイパビリティ適応 v1.0 が実装された場合に適用）

| レベル | 対象 | 実行可否 |
|--------|------|--------|
| `builtin` | リポジトリに含まれるツール | 常に実行可 |
| `shiro-generated` | Shiro 生成ツール | 登録後即座に実行可 |

ToolRegistry に登録されたツールは `workspace/tools/<name>.sh` が存在すれば即座に利用可能になる。

## 6. 外部 API 呼び出し

### 6.1 タイムアウト

| 対象 | デフォルト | 最大 |
|------|----------|------|
| HTTP リクエスト | 30秒 | 120秒 |
| LLM 呼び出し | 60秒 | 300秒 |
| ファイル操作 | 10秒 | 30秒 |

### 6.2 リトライ

| 設定 | デフォルト |
|------|----------|
| 最大回数 | 3 |
| 方式 | 指数バックオフ（1s, 2s, 4s） |
| リトライ対象 | 5xx, タイムアウト, 接続エラー |
| リトライ除外 | 4xx（バリデーションエラー） |

レート制限（429）受信時は `Retry-After` ヘッダに従う。ヘッダがない場合は60秒待機。

## 7. 禁止事項

| 禁止 | 理由 |
|------|------|
| API キーを config.yaml に平文保存 | 誤ってリポジトリに混入する可能性 |
| MaxContext（8192）を超える Ollama モデルのロード | OOM リスク |
| Chat/Worker モデルを常駐化しない使い方 | `keep_alive: -1` 必須 |
| Worker 実行ログの省略 | すべての実行には詳細ログが必須 |
| セーフガードの無効化 | Git auto-commit・保護ファイルパターンは必須 |
