## 3. 分散実行（v4.0）

### 3.1 概要

v4.0 では Agent 間通信を **SSH + JSON** で行う。設定フラグ `distributed.enabled` で切り替え。

**現状**: `distributed.enabled: false`（Claude API クレジット補充後に `true` に切り替え）

```yaml
distributed:
  enabled: false   # true にすると bmax2nd-1 への SSH 経由 Coder3 通信が有効
```

### 3.2 Transport 抽象化

```go
type Transport interface {
    Send(ctx context.Context, msg Message) (Message, error)
    Close() error
}
```

- `LocalTransport` — 同一プロセス内通信（coder1/coder2 が使用）
- `SSHTransport` — SSH 経由 JSON 送受信（coder3 が使用）

```
[fujitsu-ubunts] MessageOrchestrator
  ↓ LocalTransport（Coder1/Coder2）
  ↓ SSHTransport（Coder3）
[bmax2nd-1]       picoclaw-agent.exe → Claude API
```

### 3.3 picoclaw-agent

`cmd/picoclaw-agent/main.go` — スタンドアロンモードで動作する Agent プロセス。

| 配備先 | パス | 状態 |
|--------|------|------|
| bmax2nd-1 (Win11) | `C:\Users\nyuki\picoclaw-agent.exe` | ✅ 配備済み |

SSH 鍵: `~/.ssh/picoclaw_agent`（Ubuntu → bmax2nd-1）

## 4. 通信フォーマット

### 4.1 Agent 間 JSON メッセージ

```json
{
  "from": "Worker",
  "to": "Coder3",
  "session_id": "session_abc123",
  "job_id": "job_20260302_001",
  "message": "hello.go を作成してください",
  "context": {
    "user_request": "hello.go を作成して",
    "history": [
      {"role": "user", "content": "..."},
      {"role": "assistant", "content": "..."}
    ]
  },
  "timestamp": "2026-03-26T15:30:00Z"
}
```

### 4.2 Proposal 受け渡し（Coder → Worker）

```json
{
  "from": "Coder3",
  "to": "Worker",
  "session_id": "session_abc123",
```

## 5. Ollama 設定

### 5.1 常駐化設定

```yaml
ollama:
  keep_alive: -1          # モデルをメモリに永続化
  max_context: 8192       # MaxContext 上限（これを超えるモデルは NG）
  health_check_interval: 60  # 秒
```

### 5.2 ヘルスチェックフロー

LINE 指示ごとに以下の順で処理する:

1. LLM へリクエスト送信（事前ヘルスチェックなし）
2. ヘルスチェック常時実行（成功・失敗に関わらず）
3. Ollama が落ちていれば `ollama_restart_command` で再起動（`systemctl --user restart ollama`）
4. LLM 失敗時のみリトライ（成功時の重複応答を防ぐ）

## 6. LLM パラメータ設定

### 6.1 エージェント別パラメータ

| エージェント | 用途 | MaxTokens | Temperature |
|---|---|---|---|
| **Mio**（Chat） | 通常会話・CHAT/PLAN ルート | 512 | 0.7 |
| **Mio**（Web Search） | RESEARCH/外部検索 | 1024 | 0.3 |
| **Shiro**（Worker/フォールバック） | OPS ルート単発実行 | 4096 | 0.3 |
| **Gin**（Coder3/Claude） | CODE3 ルート | 8192 | 0.5 |

### 6.2 Worker 設定

```yaml
worker:
  auto_commit: false
  commit_message_prefix: "[Worker Auto-Commit]"
  command_timeout: 300     # 秒
  git_timeout: 30
  stop_on_error: false     # 継続モード（デフォルト）
  workspace: "/path/to/workspace"
  protected_patterns: [".env*", "*credentials*", "*.key", "*.pem"]
  action_on_protected_file: "error"
  show_execution_summary: true
```
