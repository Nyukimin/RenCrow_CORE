## 1. 構造化ログ

### 1.1 ログフォーマット

```go
logger.InfoCF("worker", "Patch execution started", map[string]interface{}{
    "job_id":         jobID,
    "route":          route,
    "patch_commands": len(commands),
})
```

**ログ出力先**: `~/.picoclaw/logs/picoclaw.log`

**ローテーション**: `~/.picoclaw/bin/log-rotate.sh`（cron 毎日 04:00）

---

## 2. 必須ログイベント

### 2.1 ルーティングイベント

| イベント | トリガー |
|---------|--------|
| `router.decision` | ルーティング決定時 |
| `classifier.error` | 分類器エラー発生時 |

### 2.2 Worker イベント

| イベント | トリガー |
|---------|--------|
| `worker.patch_parse` | Patch 解析完了 |
| `worker.execution_started` | Worker 実行開始 |
| `worker.execution_completed` | Worker 実行完了 |
| `worker.execution_failed` | Worker 実行失敗 |
| `worker.auto_commit` | Git auto-commit 実行 |
| `worker.success` | Worker 全処理成功 |
| `worker.fail` | Worker 処理失敗 |
| `worker.executed` | Worker による patch 即時実行 |

### 2.3 Coder イベント

| イベント | トリガー |
|---------|--------|
| `coder.plan_generated` | Coder による plan/patch 生成 |

---

## 3. 必須保存フィールド

すべての Worker 実行ログには以下のフィールドを含める:

| フィールド | 形式 | 説明 |
|----------|------|------|
| `job_id` | `job_{timestamp}_{random}` | Task 識別子 |
| `session_id` | `line:<user_id>` / `slack:<channel_id>` / `cli:default` | セッション識別子 |
| `initial_route` | `CHAT` / `CODE` / ... | 最初に決定したルート |
| `final_route` | `CHAT` / `CODE` / ... | 最終的に実行したルート |
| `execution_status` | `success` / `partial_success` / `failed` | 実行状態 |
| `coder_output` | plan/patch/risk の要約 | Coder の生成物 |
| `worker_result` | 成功数/失敗数、Git commit hash | Worker の実行結果 |

## 4. API キーマスキング

ログ出力時に API キーを自動マスク:

```
ANTHROPIC_API_KEY=sk-ant-1234... → ANTHROPIC_API_KEY=sk-ant-****
OPENAI_API_KEY=sk-...            → OPENAI_API_KEY=sk-****
DEEPSEEK_API_KEY=...             → DEEPSEEK_API_KEY=****
```

実装: `pkg/logger/masking.go`

## 5. Viewer / EventHub

### 5.1 Viewer SSE イベント

```
IdleChat → EventHub.OnEvent() → SSE broadcast → viewer.html
```

ルート色の凡例（Viewer Timeline 表示）:

| ルート | 色 |
|--------|-----|
| CHAT | デフォルト |
| CODE / CODE1-3 | 緑系 |
| OPS | 橙 |
| IDLECHAT | 紫 |

### 5.2 包括的ログ実装

全 Agent 操作を `~/.picoclaw/logs/picoclaw.log` に記録。
Viewer / EventHub / Orchestrator の全フローを追跡可能。

## 6. セッション管理とライフサイクル

### 6.1 session_id 管理

チャネル単位で管理:

```
LINE:  "line:<user_id>"
Slack: "slack:<channel_id>"
CLI:   "cli:default"
```

### 6.2 日次カットオーバー

毎日 0 時に `ResetSession()` を呼び、以下を実行:

1. セッションリセット（RAM / Redis の短期記憶をクリア）
2. `daily_note` に保存
3. `LightMemory.ClearAll()`（AgentPersona v1.0 実装後）

### 6.3 短期記憶（short_memory）

会話履歴の要約を保存（日次カットオーバーでリセット）。
詳細は [§6 会話エンジン](06_会話エンジン.md)を参照。

---

## 7. Obsidian 連携

構造化ログを Obsidian 形式で出力可能（設定で有効化）。

```yaml
logging:
  obsidian:
    enabled: true
    vault_path: "/path/to/obsidian/vault"
    daily_note_template: "picoclaw-{date}.md"
```
