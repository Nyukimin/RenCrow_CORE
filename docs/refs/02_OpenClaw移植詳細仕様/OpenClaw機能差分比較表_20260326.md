**OpenClaw**: https://github.com/openclaw/openclaw（TypeScript/Node 24+、MIT）

**比較基準**: `docs/01_正本仕様/` の現行仕様（feature/rencrow ブランチ）および上記 README / ドキュメント（2026-03-26 時点）  
**凡例**: ✓ 実装済み　△ 部分実装/相当機能あり　✗ 未実装

### NanoClaw / RenCrow の位置づけ（一文）

| プロダクト | 一言 |
|-----------|------|
| **NanoClaw** | [qwibitai/nanoclaw](https://github.com/qwibitai/nanoclaw) は **小さな TypeScript 本体＋Claude Agent SDK** と **Linux コンテナ隔離**を前面に出し、**スキル（`/add-*`）でチャネル追加**する思想。設定ファイル肥大を避け **コード/Claude Code でカスタム**する。 |
| **RenCrow** | [sipeed/rencrow](https://github.com/sipeed/rencrow) は **Go 単一バイナリ・低メモリ**と **30+ LLM / 17+ チャネル / ネイティブ MCP / ClawHub スキル**を README レベルで揃えた **汎用パーソナルアシスタント**（OpenClaw フォークではないと明記）。 |

## RenCrow 独自機能（OpenClaw にない）

| 機能 | 説明 |
|------|------|
| IdleChat | アイドル時エージェント自律雑談（通常/未来展望/ストーリー）|
| /local, /cloud コマンド | ルートごとのクラウド LLM 禁止フラグ |
| /code〜/code4 コマンド | Coder 明示ルーティング（4 スロット: 仕様/実装/レビュー/実験） |
| VTuber ブリッジ | VTuber 連携 |
| Ollama 常駐管理（keep_alive=-1）| Ollama モデルの常駐 + 自動再起動 |
| Obsidian 連携 | ログの Obsidian 形式出力 |
| Worker patch 即時実行 | Coder が生成した plan/patch を Worker が即座に実行 |

## NanoClaw・RenCrow の突出点（上表の凡例と併読）

| プロダクト | 差分表で「OpenClaw 相当」とは別物になりやすい点 |
|-----------|-----------------------------------------------|
| **NanoClaw** | **OS レベルに近いコンテナ隔離**（Docker / Apple Container / Docker Sandboxes）。**生 API キーをエージェントが持たない**設計（Agent Vault 経由）。**設定ファイル非増殖**（フォーク＋Claude Code で改変する思想）。 |
| **RenCrow** | **組み込み・エッジ向け**（公式 README の省メモリ・単一バイナリ・多アーキテクチャ）。**MCP ネイティブ**、**ClawHub**、**多数のチャネル・プロバイダを設定で列挙**する方向（NanoClaw の「スキルだけ足す」思想とはトレードオフが異なる）。 |
