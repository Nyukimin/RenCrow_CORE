# 機能仕様

## 会話とルーティング

通常メッセージの route owner は Mio です。処理順序は、明示 command、Mio の `DecideAction`、rule dictionary、classifier、決定 route の実行です。無効な判断や利用不能な runtime は、契約で定めた failure または CHAT fallback として扱います。

Viewer の通常会話は `to=mio|shiro|kuro|midori` を受け付けます。明示 command は会話相手の選択より優先されます。選択した runtime が利用不能な場合、Mio へ黙って切り替えません。

## Worker、Coder、Advisor、Tool

- Worker は決定済み route の実行、状態、結果、side effect、承認境界を所有する。
- Coder は設計、差分、提案を返し、外部 side effect の最終責任者にならない。
- Advisor は専門的な助言を返し、採否と実行責任を持たない。
- Tool は明示された capability を実行し、人格、長期記憶、最終意思決定を持たない。

## Memory、Recall、Knowledge

- 会話履歴は発言ごとの `from` と `to` を保持する。
- RecallPack は複数の検索結果を budget、role visibility、provenance とともに組み立てる。
- Knowledge Relation は Entity、Topic、Project を制限付き 1-2 hop で横断し、採用理由を trace に残す。
- 外部検索結果、Advisor 出力、収益候補は、人間確認なしに確定 Knowledge へ昇格しない。

## 継続作業と承認

Backlog、Scheduler、Heartbeat、Workstream が長時間作業を管理します。外部公開、送信、請求、契約、価格決定などの high-impact action は approval gate を通過するまで draft に留めます。

## Economic Objective

Opportunity、EconomicTask、RevenueEvent、EconomicReflection を管理します。Heartbeat の discovery は候補の draft 作成までに制限し、外部実行を自動化しません。

## Viewer と観測性

Viewer は会話、agent、job、log、workstream、approval、memory、advisor、economic state を REST/SSE で表示します。raw trace は要約と詳細を分け、`unavailable`、`pending`、`degraded`、`error` を成功表示へ丸めません。

## 音声と外部 runtime

CORE は STT/TTS の接続、request、Viewer 操作、状態を持ちます。表示文生成、音声合成、音声取得、ブラウザ再生は別々の成功条件です。LLM も inference endpoint と management endpoint を分け、ready 確認後に選択を確定します。
