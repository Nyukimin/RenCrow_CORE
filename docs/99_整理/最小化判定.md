# 最小化判定

## 残す基準

この最小セットでは、次のどれかを満たすものだけ残しました。

1. 正本である。
2. 起動・runtime・LLM/TTS/STT経路を誤らないために必要である。
3. 現行キャラクターの責務と安全境界を理解するために必要である。
4. 記憶・検索・RAG・外部取得の設計判断に必要である。
5. 実装順、受入基準、構築リスクの判断に必要である。

## 外す基準

次の資料は、元ZIPには残っていますが、最小セットからは外しました。

- 旧仕様アーカイブ。
- v2分割仕様。正本へ統合済みか確認対象であり、入口にはしない。
- 監査差分詳細。必要時のみ戻す。
- AI提案資料。採用状態が正本に反映されていないものは入口にしない。
- 日付付き作業ログ。完了済み内容は正本へ反映し、最小版には置かない。
- IdleChat 詳細仕様群。正本 `02_正本仕様/04_IdleChat.md` を入口にし、詳細実装時だけ戻す。
- STT/TTS 詳細仕様。Runtime の境界理解には正本 Config を残し、音声実装時だけ戻す。
- codebase-map 詳細解析。入口には `05_運用/02_Codebase_Run_Summary.md` だけ残す。

## 戻す条件

- IdleChat の特定カテゴリを実装する → `09_IdleChat仕様/` から該当詳細を戻す。
- STT/TTS を実装・修正する → `STT_TTS/` から現行仕様だけ戻す。`old/` は戻さない。
- コードの影響範囲を詳しく見る → `codebase-map/` の該当 module だけ戻す。
- プロンプト資産を更新する → `workspace/persona/` を正本にし、docs 側の古い prompt メモは必要な差分だけ吸収する。
- 旧仕様との差分確認が必要 → `04_監査差分分析/` を一時的に戻す。

## 初回レビュー後の復帰判断

2026-07-01 の初回レビューでは、元 `docs.zip` からすぐ恒久復帰する資料はないと判断した。

ただし、次の未解決論点は元 `docs.zip` の詳細資料を一時復帰して確認する可能性がある。

- 承認フロー存廃。
- `risk=high` 時の停止ルール。
- サニタイズ責務の場所。

これらは `00_引き継ぎ/DOCチケット_初回未解決論点.md` と個別チケットに分離した。資料を戻す場合は、戻したファイル名、理由、関係する正本、一時参照か恒久追加か、いつ外すかをチケットへ記録する。

## この最小版の弱点

- すべての詳細実装手順を含んでいない。
- 履歴の完全性は元ZIPに依存する。
- 生成した要約ドキュメントは正本ではない。

その代わり、通常の理解・運用・構築判断では、まずこのセットだけを見れば迷わない構成にしています。
