## 採用済み・未実装

### 1. Advisor production score loop

Advisor run の採否・成果 writer と、Scheduler/Heartbeat からの日次 score snapshot 集計を production loop に接続する必要があります。二重採用、遅延 outcome、空 window を含む idempotent な contract test が必要です。

### 2. Economic Artifact chain

Opportunity から Workstream Goal までは接続済みです。Opportunity、Goal、Artifact、Approval、Delivery、RevenueEvent、Reflection を同じ provenance/trace で結ぶ production use case は未完了です。公開や請求は引き続き承認前に実行しません。

### 3. Feature ownership migration

一部 feature は legacy-body で稼働し、`internal/features/*` の registrar は境界だけを持ちます。既存 API、provider、store、CLI、Viewer、background job を保ったまま、route/job ごとに移管します。

### 4. Legacy live/lab Viewer cleanup

外部利用の正本は`RenCrow_PORTAL`へ移りました。CORE内に残る従来の`mode=live|lab`資産は、PORTALの実ブラウザ機能同等性と配布導線を確認するまで互換経路として保持します。削除は別リリースで行い、Debug Viewerのtimeline、SSE、音声、内部運用画面を巻き込まない契約テストを先に固定します。

### 5. Interaction client profileの統一

COREのChat／IdleChat API、PORTALのWeb操作、CMDのChatは実装済みです。PORTAL、CMD、
ASSISTANTを同じInteraction意味論で扱う方針は採用済みですが、CMDの
`idlechat watch|start|stop`、ASSISTANT runtime／Device delivery、profile名を使う統一認可は
未実装です。現行allowlistと認可を維持したままcontract testを揃え、実caller間に重複が
確認されるまでは共通SDKを先行作成しません。

## Deployment またはライブ検証に依存するもの

外部 LLM/STT/TTS/Vision/GAMES runtime の起動、実ブラウザの responsive 表示、実音声再生は環境依存です。コード実装済みでも、各 deployment では health、実 request、実表示・再生を別々に検証してください。

## 現在仕様に含めない旧構想

- 旧 `pkg/*` package path を前提にした architecture snapshot
- 旧 `/deliberate` と agent heartbeat protocol
- Advisor、Knowledge、Economic、To-Be Ops を全面未接続とする古い状態記述
- 無審査の memory promotion、承認なしの公開・送信・請求
- CORE が外部 module の実装本体を再び所有する構成

これらは現在の未実装 backlog ではありません。再採用する場合は、現行の Mio/Shiro/Coder/Advisor 境界、安全条件、cost、timeout、trace を先に再定義します。
