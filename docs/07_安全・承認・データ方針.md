# 安全・承認・データ方針

## 承認が必要な操作

RenCrow_CORE は自律処理を draft-only から開始します。次の操作は human approval なしに実行しません。

- 外部への公開・投稿・送信
- 請求、支払、契約、価格決定
- production data や重要ファイルの破壊的変更
- credential、権限、network boundary の変更
- 高リスク Tool/Browser action

Approval には scope、requester、対象 artifact、期限、判断、実行結果を記録し、同じ承認を別目的へ流用しません。

## Agent と Tool の境界

Advisor と Coder は提案者であり、実行責任者ではありません。Tool は人格や判断主体ではなく、allowlist、sandbox、timeout、監査 policy の下で呼び出される capability です。Shiro/Worker が side effect と承認境界の責任を持ちます。

## Knowledge と Memory

- external search は発見であり、確定 Knowledge の証拠ではない。
- source、取得時刻、要約、採用理由を provenance として保持する。
- Advisor 出力、候補資料、Revenue draft を自動 promotion しない。
- role visibility と context budget を適用し、不要な個人情報を Recall しない。
- 誤りを訂正できるよう、原資料と派生要約を区別する。

## Secret と個人データ

- secret を source、docs、log、trace、artifact に保存しない。
- local config、DB、session、音声、画像、browser profile は repository の外で管理する。
- log には必要最小限の識別情報だけを残し、公開前に redaction する。
- archive branch が公開であることを前提に、secret や私的 runtime data は archive にも置かない。

## 失敗と degraded state

provider、LLM、STT、TTS、browser、storage の失敗を空の成功へ変換しません。`unavailable`、`pending`、`degraded`、`error` を区別し、再試行、fallback、利用者への表示を契約として扱います。
