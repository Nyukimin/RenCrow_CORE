# [DOC] サニタイズ責務の所在を確認する

## 背景

`05_運用/02_Codebase_Run_Summary.md` では、サニタイズ責務が `pkg/security` 側か `materials` 側か未解決の可能性が示されている。責務境界は安全性とログ/素材管理に関わるため、推測で正本へ追記しない。

## 対象ファイル

- `02_正本仕様/01_仕様.md`
- `02_正本仕様/02_実装仕様.md`
- `05_運用/02_Codebase_Run_Summary.md`
- `99_整理/除外ファイル一覧.csv`

## 作業内容

- [ ] 正本内のセキュリティ、データ取り扱い、素材管理の記述を確認する。
- [ ] 実装側のサニタイズ処理の所在を確認する。
- [ ] `pkg/security` と `materials` のどちらが責務を持つべきか、または分担すべきかを整理する。
- [ ] 判断材料が足りない場合だけ、元 `docs.zip` から監査差分または codebase-map の該当 module を一時復帰する。

## 正本への影響

- 正本変更ありの可能性あり。
- 変更する場合: `02_正本仕様/01_仕様.md` または `02_正本仕様/02_実装仕様.md`

## 派生文書への影響

- `04_構築指標/02_受入基準・テスト観点.md`
- `05_運用/02_Codebase_Run_Summary.md`

## 元 docs.zip から戻す資料

- あり。必要時のみ一時復帰。
- 候補:
  - `04_監査差分分析/仕様_実装仕様_対応表.md`
  - `04_監査差分分析/仕様_実装仕様_差分メモ.md`
  - `codebase-map/modules/infra.md`
  - `codebase-map/modules/agent.md`

## 完了条件

- [ ] サニタイズ責務の所在が正本・実装の根拠付きで説明できる。
- [ ] sensitive data、ログ、素材管理の境界が混同されていない。
- [ ] 受入基準へ追加する場合は正本変更後に行う。

## リスク

- サニタイズを複数箇所へ重複実装して責務が曖昧になる可能性。
- ログや素材に sensitive data が残るリスクを過小評価する可能性。
