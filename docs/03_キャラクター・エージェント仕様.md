# キャラクター・エージェント仕様

RenCrow では、画面上の character、内部 agent、外部 Advisor、実行 Tool を別の責務として扱います。

| 主体 | 主な責務 | side effect の最終責任 |
| --- | --- | --- |
| Mio | 通常入力の受領、route decision、会話、最終 user response | 原則として持たない |
| Shiro | Worker/OPS の実行責任、承認境界、Advisor 助言の採否 | 持つ |
| Kuro / Heavy | 深い分析を必要とする chat route | 持たない |
| Midori / Wild | 発想、創作、探索的な chat route | 持たない |
| Coder1/2/3 | 設計、差分、実装提案 | 持たない |
| Advisor | 外部専門家としての助言 | 持たない |
| Tool | 限定された capability の実行 | 呼出元の policy に従う |

## 会話相手と実行 role

Viewer で Shiro、Kuro、Midori を選んでも、通常は会話 route として扱います。Shiro との会話と Shiro が担う OPS execution は同一ではありません。明示 command がある場合だけ、対応する command route を優先します。

Agent間で作業を移譲する場合は、移譲元の指名付き依頼、移譲先の指名付き復唱、移譲先の指名付き完了報告を共通Chatで行う。詳細は「機能仕様 / Agent間の口頭移譲」を正本とする。

## Persona と共有状態

- Persona は character ごとの話し方、役割、制約を定義する。
- 会話履歴、過去のエピソード、確定済みのユーザー記憶は全character・Agentで共有する。記憶のアクセスをPersonaやroleごとに分割しない。
- 想起したエピソードはAgentの非表示コンテキストとし、Agentが実際に発言した場合のみ通常のassistant発言として共通会話履歴に残す。
- 内部 reasoning や route detail を user-facing response や TTS 文面へ混ぜない。
- AgentProfile は capability と AutonomyEnvelope を表し、許可範囲を越える実行を認めない。

## Workspace shared control

portableなcharacter制御は`workspace_dir/control/`の4ファイルを一つのversioned setとして読み込みます。

| file | 責務 |
| --- | --- |
| `agents.yaml` | characterごとのrole、capability、非担当領域 |
| `routing.yaml` | CORE route categoryと実行owner、`CHAT` fallback |
| `handoff.yaml` | 全character共通の担当外返却contract |
| `tools.yaml` | Tool選択姿勢、provider preference、自動fallback禁止 |

character固有Promptは`00_system.md`、`10_policy.md`、`20_scope.md`、`30_knowledge.md`だけを持ち、共通routing、handoff、Tool制御を重複保持しません。COREは4ファイルを一括検証し、対象characterの制御blockをSystemPromptへ付加します。4ファイルの一部欠落、route ownerとproduction wiringの不一致、agentによるhandoff先決定、Toolの自動fallbackはconfig load errorです。

Workspace controlは権限付与やdeployment設定ではありません。Toolの実在、引数schema、endpoint、secret、side effect許可、approvalはCORE runtimeとToolRunner metadataを正本とし、Workspace記述で拡張できません。

## SystemPrompt の現在時刻

character応答を生成するLLM requestでは、WorkspaceのPrompt本文と共有controlを組み立てた後、正本のSystemPrompt末尾へ次の形式でrequest時点のJST現在時刻を付加します。

```text
現在時刻（JST）: 2026-07-24 22:45:12 JST
```

対象はMio、Shiro、Kuro / Heavy、Midori / Wild、Coder1〜4、およびIdleChatのcharacter応答です。時刻は保存済みPromptへ書き込まず、LLM送信直前に生成するため、service起動時刻へ固定されません。classifier、品質判定、要約など、character応答ではない内部LLM処理は対象外です。

全LLM共通の日時middlewareは、SystemPromptにこのJST行が存在する場合は日時を重複注入しません。character応答ではない内部LLM処理へ日時を補う場合も、同じJST形式を使用します。

ConversationEngineから共有するUserProfile、会話履歴、要約、Knowledgeはcharacter間で維持します。一方、保存済みRecallPackのPersona SystemPromptは別characterの固有SystemPromptへ重ねません。Mio以外のcharacter応答では、選択中characterのPromptだけを人格の正本とします。

## 失敗時の契約

指定した role や runtime が利用不能な場合は、その対象の error/degraded state を返します。別 character への無言 fallback は行いません。
