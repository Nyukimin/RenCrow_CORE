# キャラクター・エージェント仕様

RenCrow では、画面上の character、内部 agent、外部 Advisor、実行 Tool を別の責務として扱います。

| 主体 | 主な責務 | side effect の最終責任 |
| --- | --- | --- |
| Mio | 通常入力の受領、route decision、会話、最終 user response | 原則として持たない |
| Shiro | Worker/OPS の実行責任、承認境界、Advisor 助言の採否 | 持つ |
| Kuro / Heavy | 深い分析を必要とする chat route | 持たない |
| Midori / Wild | 発想、創作、探索的な chat route | 持たない |
| Coder1/2/3 | 設計、差分、実装提案 | 持たない |
| Advisor | 外部専門家としての助言 | 持たない |
| Tool | 限定された capability の実行 | 呼出元の policy に従う |

## 会話相手と実行 role

Viewer で Shiro、Kuro、Midori を選んでも、通常は会話 route として扱います。Shiro との会話と Shiro が担う OPS execution は同一ではありません。明示 command がある場合だけ、対応する command route を優先します。

## Persona と共有状態

- Persona は character ごとの話し方、役割、制約を定義する。
- 会話履歴は character 間で共有できるが、Recall には role visibility を適用する。
- 想起したエピソードはAgentの非表示コンテキストとし、Agentが実際に発言した場合のみ通常のassistant発言として共通会話履歴に残す。
- 内部 reasoning や route detail を user-facing response や TTS 文面へ混ぜない。
- AgentProfile は capability と AutonomyEnvelope を表し、許可範囲を越える実行を認めない。

## 失敗時の契約

指定した role や runtime が利用不能な場合は、その対象の error/degraded state を返します。別 character への無言 fallback は行いません。
