# RenCrow CORE 仕様分類の敵対的検証ログ

## 概要

「未実装」と「古い記述」を取り違えないため、各所見を反証する前提で現行コード、production wiring、対象 test、仕様日付を照合した。全9所見の事実部分は confirmed だが、それぞれが同じ種類の backlog を意味するわけではない。

| verdict | 件数 |
| --- | ---: |
| confirmed | 9 |
| refuted | 0 |
| uncertain | 0 |

## 所見

### DEV-001: active 文書に旧 `pkg/*` path が混在する

- Claim: 現行正本 child に存在しない旧 path と旧 routing ownership が残る。
- Verdict: `confirmed`
- Lens: correctness, spec-consistency, reproducibility
- Evidence:
  - `docs/02_正本仕様/02_実装仕様/01_エージェント・責務境界.md:11-15` は旧 architecture と明記する。
  - 同文書 `:86-88` はなお `pkg/agent/loop.go` と `pkg/agent/context.go` を現行説明に使う。
  - `docs/02_正本仕様/02_実装仕様/06_参照・実装状況.md:11,51-75` は snapshot と旧 path を含む。
- Refutation attempt: 現行 tree か互換 shim に同 path が残る可能性を検索したが、現在の production owner を示す証拠にはならなかった。
- Classification impact: `stale_snapshot`。未実装 backlog にはしない。

### DEV-002: Go version guidance が `go.mod` と一致しない

- Claim: `CLAUDE.md` は Go 1.23、`go.mod` は Go 1.25.0 を指定する。
- Verdict: `confirmed`
- Lens: correctness, spec-consistency
- Evidence: `CLAUDE.md:46-49`, `go.mod:1-3`
- Refutation attempt: toolchain override や別 module root の可能性を除外し、同 repo root のファイルを比較した。
- Classification impact: 文書陳腐化。機能未実装ではない。

### NI-001: 13 feature registrar は ownership boundary の予約だけである

- Claim: agent/avatar/backlog/chat/core/distributed/heartbeat/repair/revenue/scheduler/security/worker/workstream の registrar は route/background 実装を持たない。
- Verdict: `confirmed`
- Lens: correctness, spec-consistency, reproducibility
- Evidence: 各 `internal/features/<name>/registrar.go` は25行で、`RegisterRoutes` は引数を捨て、`StartBackground` は `nil` を返す。
- Refutation attempt: feature 外の production code と composition root を検索したところ、機能本体は legacy-body に存在した。
- Classification impact: 挙動は `implemented_legacy_body`、ownership migration だけ `unimplemented_confirmed`。

### ERR-001: signal 受信時の HTTP drain がない

- Claim: server shutdown を待たず `os.Exit(0)` する。
- Verdict: `confirmed`
- Lens: concurrency, correctness, reproducibility
- Evidence: `cmd/rencrow/main.go:91-97`、repo 内に `server.Shutdown` の production call がない。
- Refutation attempt:別 goroutine、defer、wrapper server に drain がないか検索したが確認できなかった。
- Classification impact: quality gap。正本採用要件と確認できないため機能未実装とは分離。

### INCONSIST-001: Ops registrar が複数 feature の route を横断所有する

- Claim: `internal/features/ops` が status/jobs/backlog/scheduler/workstream/revenue/advisor/profile 等を登録する。
- Verdict: `confirmed`
- Lens: correctness, spec-consistency
- Evidence: `internal/features/ops/registrar.go:67-109`
- Refutation attempt: 専用 registrar から委譲される構造か確認したが、13 registrar は予約状態だった。
- Classification impact: 非破壊 modularization の途中状態。CORE-GAP-003 に統合。

### TEST-001: `cmd/rencrow-agent` に local test がない

- Claim: production Go files はあるが同 directory の `_test.go` と他 test からの直接参照がない。
- Verdict: `confirmed`
- Lens: correctness, reproducibility
- Evidence: `cmd/rencrow-agent` の production files と repo-wide test reference 検索。
- Refutation attempt: package 外 test、E2E、build-only coverage を検索したが直接 coverage は確認できなかった。
- Classification impact: test gap。機能未実装とは分離。

### STATE-001: 2026-07-14 の派生資料は接続状態が古い

- Claim: Advisor、Knowledge、Economic、To-Be Ops を未接続とする記述は現在コードと一致しない。
- Verdict: `confirmed`
- Lens: correctness, spec-consistency, reproducibility
- Evidence:
  - 派生資料の最終 commit: `d0e6623`、2026-07-14。
  - 対応実装の最終 commit: `3a14287`、2026-07-15。
  - `cmd/rencrow/runtime_advisor.go`, `runtime_conversation.go`, `runtime_heartbeat.go`, `runtime_dependencies.go`
  - `internal/adapter/viewer/assets/js/tabs/to-be-ops.js`
- Refutation attempt: 型や未接続 handler だけではないか確認し、composition root、Viewer route、target tests まで追跡した。
- Classification impact: 実装済み。派生資料の status 更新が必要。

### NI-002: Advisor adoption writer と日次 score 集計が production 未接続

- Claim: store/schema/builder はあるが、採否 writer と日次 snapshot job がない。
- Verdict: `confirmed`
- Lens: correctness, spec-consistency, reproducibility
- Evidence:
  - store: `internal/infrastructure/persistence/advisor/*_store.go`
  - builder: `internal/application/advisor/recording.go:123`
  - production 外の call: store tests と `test/e2e/browser/seed_to_be_ops/main.go`
  - 要求: `docs/02_正本仕様/11_RenCrow_ToBe_統合実装仕様.md:211-213,353-356`
- Refutation attempt: `SaveAdvisorAdoption`、`SaveAdvisorScoreSnapshot`、`BuildScoreSnapshot` の全参照を検索した。
- Classification impact: CORE-GAP-001。

### NI-003: Economic acceptance の Artifact 接続がない

- Claim: Opportunity から Workstream Goal は作れるが、Artifact を作る production use case はない。
- Verdict: `confirmed`
- Lens: correctness, spec-consistency, reproducibility
- Evidence:
  - `internal/application/revenue/economic.go:203-229` は Goal の verification に `artifact exists` を入れるだけ。
  - `internal/adapter/viewer/revenue_economic_handler.go` は Opportunity → Goal を接続する。
  - revenue application/handler に Artifact create/save call はない。
  - 要求: `docs/02_正本仕様/10_RenCrow_ToBe_統合仕様/05_Economic_Objective・Revenue.md:158-167`、統合実装仕様 `:363-365`。
- Refutation attempt: Workstream domain と一般 Artifact APIs の存在を確認したが、Economic chain からの関連付けは見つからなかった。
- Classification impact: Economic MVP は実装済み、end-to-end acceptance の不足だけ CORE-GAP-002。

## 結論

現在の採用済み未実装は、Advisor score production loop、Economic Artifact chain、feature ownership migration の3群に絞れる。古い `pkg/*` architecture、旧 Heartbeat/Deliberation、派生資料の「後続」注記、外部 runtime 本体は、この3群へ混ぜない。
