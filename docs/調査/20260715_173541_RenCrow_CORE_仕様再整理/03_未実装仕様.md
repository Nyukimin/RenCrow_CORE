# 未実装仕様

## 概要

本章には「最新仕様で採用済み」かつ「production 経路が存在しない、または acceptance chain が閉じていない」項目だけを載せる。古いチェックリスト、外部 module の本体、ライブ未確認、単なるテスト不足は未実装仕様へ混ぜない。

## 採用済み・未実装

| ID | 優先度 | 未実装範囲 | 既にあるもの | 完了条件 |
| --- | --- | --- | --- | --- |
| CORE-GAP-001 | 高 | Advisor の採用結果を production で記録し、日次 score snapshot を生成する接続 | run/policy/adoption/score の型・store・score builder・read API | 採否/成果 writer、定期集計、snapshot 保存、失敗監視、テスト |
| CORE-GAP-002 | 高 | Opportunity から Artifact を作り、Economic chain 全体を同一 trace で閉じる接続 | Opportunity、Workstream Goal、Artifact domain、Approval、RevenueEvent、Reflection は個別に存在 | Opportunity → Goal → Artifact → Approval → RevenueEvent → Reflection の contract/E2E |
| CORE-GAP-003 | 中 | 13 feature の route/background ownership を予約 registrar へ移管 | 対象機能の legacy-body と production wiring | 挙動を変えず registrar が所有し、旧 wiring を除去、contract tests 通過 |

## CORE-GAP-001: Advisor adoption と日次 score 集計

最新 To-Be は Advisor score の更新と日次集計を要求している。現状は次まで実装済みである。

- `SaveAdvisorAdoption` と `SaveAdvisorScoreSnapshot` を持つ JSONL/SQLite store
- advice run と policy decision の記録
- `BuildScoreSnapshot` の計算ロジックと unit test
- score snapshot の Viewer read API

一方、production code から `SaveAdvisorAdoption` を呼ぶ採否・成果 feedback writer、および `BuildScoreSnapshot` を定期実行して `SaveAdvisorScoreSnapshot` へ保存する scheduler は見つからない。browser seed とテストによる直接保存は production 接続とは数えない。

必要な実装境界:

1. Shiro/approval/workstream の確定結果から AdvisorRunID と adoption/outcome を関連付ける。
2. provenance と人間採否を保持して adoption record を保存する。
3. Scheduler/Heartbeat から日次 window の集計 job を idempotent に実行する。
4. 失敗、対象件数、window、snapshot ID を監査可能にする。
5. 同一 run の二重採用、遅延 outcome、空 window を contract test する。

## CORE-GAP-002: Economic chain の Artifact 接続

最新 To-Be の acceptance は、Opportunity から Workstream / Artifact / Approval / RevenueEvent / Reflection まで接続できることを要求する。

現在は Opportunity から `GoalFromOpportunity` を使って Workstream Goal を保存できる。しかし生成される Goal の verification に文字列 `artifact exists` が入るだけで、Opportunity/Goal から Artifact を生成・関連付ける production use case はない。Artifact domain と一般の Workstream API が存在することは、この接続が完成している証拠にはならない。

必要な実装境界:

1. Opportunity と Workstream/Goal/Artifact の ID、provenance、expected profit を関連付ける。
2. Artifact は draft で作成し、外部公開前に approval gate を必須化する。
3. delivery、RevenueEvent、Reflection まで同じ trace を辿れるようにする。
4. 途中失敗の再実行を idempotent にし、勝手な公開・請求・送信を行わない。
5. handler 単体ではなく chain 全体の contract/E2E を追加する。

## CORE-GAP-003: feature ownership 移行

次の13 feature registrar は、route と background boundary を予約するだけである。

```text
agent
avatar
backlog
chat
core
distributed
heartbeat
repair
revenue
scheduler
security
worker
workstream
```

各機能の多くは legacy-body で実装済みなので、これは機能開発ではなく ownership migration である。移管は route/background job の小さい単位で行い、既存 API、provider、store、CLI、Viewer、job behavior を削除しない。

推奨順序:

1. contract test で現在の route/status/side effect を固定する。
2. registrar の `Dependencies/Ports` を最小化する。
3. composition root の登録を feature registrar へ移す。
4. 同一 route の二重登録と background job の二重起動を防ぐ。
5. legacy wiring を削除し、挙動等価性を確認する。

## 品質是正候補（採用仕様の未実装とは別）

| ID | 状態 | 内容 | 扱い |
| --- | --- | --- | --- |
| QUALITY-001 | confirmed | SIGINT/SIGTERM 後に HTTP server の `Shutdown`/drain を行わず `os.Exit(0)` する | graceful shutdown を正本要件として採用するなら仕様化して実装 |
| QUALITY-002 | confirmed | `cmd/rencrow-agent` の production files に対応する local test がない | risk と保守方針に応じて contract/unit test を追加 |
| QUALITY-003 | confirmed | `internal/features/ops` が backlog/scheduler/workstream/revenue/advisor/profile 等を横断所有する | CORE-GAP-003 の移管時に責務を狭める |

これらは実装品質や設計負債として重要だが、現在の正本 acceptance に直接書かれた機能不足とは限らない。そのため上の3件とは別枠にした。

## 未実装に数えない項目

| 項目 | 理由 |
| --- | --- |
| Knowledge relation importer / batch / 1-2 hop recall | 2026-07-15 の production wiring とテストで実装済み |
| Economic Opportunity discovery scheduler | Heartbeat に draft-only、idempotent な接続がある |
| Advisor/Knowledge/Economic の Viewer 表示 | handler、client、To-Be Ops JS が実装済み |
| full Knowledge backfill | 即時 full backfill をしないことが最新仕様の方針 |
| `modules/advisor` / `modules/economic` | adapter/contract 安定前は公開 module にしない設計。存在しないこと自体は gap ではない |
| RenCrow_LLM/TTS/STT/Vision/GAMES の処理本体 | 別 repo の ownership。CORE は接続契約を持つ |
| 旧 `/deliberate` と agent heartbeat protocol | 現行責務と競合するため、実装前に再採用判断が必要 |
| ライブ browser/audio/runtime の今回未確認 | 検証未実施であり、コード未実装の証拠ではない |
