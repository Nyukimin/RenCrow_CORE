# 実装済み仕様

## 概要

本章は、production code の composition/wiring と対象テストから実装を確認できた仕様をまとめる。`internal/features/*` への移管前でも、現行 `cmd/rencrow` と legacy-body が接続されている機能は実装済みとして記載し、ownership 移行だけを未実装へ分離する。

## 1. Agent と routing

- 通常 message の route decision は Mio が所有する。
- explicit command、rule dictionary、classifier、CHAT fallback の経路がある。
- Viewer の相手選択 `mio|shiro|kuro|midori` は server 側で role に解決される。
- Shiro は chat role と OPS execution role を分離して扱う。
- runtime が使えない場合の暗黙 Mio fallback を避ける契約がある。
- Coder は proposal を返し、実 side effect と承認責任は Worker 側に残る。

## 2. Memory と Knowledge Relation

- conversation persistence、RecallPack、vector/DuckDB を含む検索経路が存在する。
- relation store は runtime 起動時に接続される。
- config 有効時の import hook が新規 item の relation を生成する。
- `rencrow knowledge relations build` が dry-run と batch build を提供する。
- runtime expansion は 1-2 hop に clamp され、reason を trace に追加する。
- Viewer と `pkg/rencrowclient` から relation/recall を照会できる。

したがって、2026-07-14 の派生資料にある「importer / batch / 1-2 hop runtime expansion は後続」という状態記述は、2026-07-15 時点では解消済みである。full backfill は別 job とする明示的方針であり、通常起動時の必須機能ではない。

## 3. Advisor と AgentProfile

実装済みの範囲:

- JSONL / SQLite の Advisor store
- Codex advisor registry と `AdvisorService`
- Shiro への AdvisorService / policy service injection
- advice run と policy decision の記録
- AgentProfile catalog と AutonomyEnvelope
- Advisor runs、scores、profiles、policy の Viewer read APIs
- unavailable 時に空成功へ見せない status/warning response

Advisor を直接 side-effect owner にせず、Shiro が助言の採否と実行責任を持つ境界も実装に反映されている。

## 4. Economic Objective

実装済みの範囲:

- Opportunity の登録、正規化、expected profit の扱い
- EconomicTask の draft と task kind 別 human approval 判定
- EconomicReflection の作成
- RevenueEvent から Reflection を作る use case と Viewer API
- Opportunity から Workstream Goal を作る use case と Viewer API
- Heartbeat による idempotent な Opportunity discovery draft
- opportunities/tasks/reflections の persistence、Viewer API、client validation
- 公開・送信・請求等を自動実行せず draft/approval に留める guard

Economic Objective 全体を「未実装」とするのは誤りである。ただし、最新 acceptance の end-to-end chain は一部未完了であり、[未実装仕様](03_未実装仕様.md) に分離した。

## 5. Viewer / To-Be Ops

- Advisor run/score/profile、Knowledge relation、Opportunity、EconomicTask、Reflection の endpoint が runtime に登録される。
- To-Be Ops tab はそれらを card として取得・表示する JavaScript を持つ。
- browser seed と E2E 用経路がある。
- Go handler/client の対象テストは通過した。

今回、実ブラウザで responsive/mobile 表示は再実行していない。このためコード実装は確認済みだが、現在のライブ画面品質は `runtime_unverified` とする。

## 6. feature facade と legacy-body

feature facade/registrar は31領域に存在し、routes を移管済みの feature と、境界だけ予約した feature が混在する。空 registrar の領域でも、chat、worker、workstream、revenue、heartbeat 等の本体は `internal/application`、`internal/adapter`、`internal/infrastructure`、`cmd/rencrow` に残って production wiring されている。

この状態は「機能未実装」ではなく「モジュール所有権移行中」である。削除や一括移動ではなく、route/background job ごとに contract test を維持して移管する。
