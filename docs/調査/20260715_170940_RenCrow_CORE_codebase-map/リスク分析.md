## 影響波及図

### ERR-001 Signal shutdownの影響伝播

~~~mermaid
flowchart TD
  SIG[SIGINT or SIGTERM] --> CLOSE[Dependencies close]
  SIG --> EXIT[os.Exit]
  HTTP[In-flight HTTP or SSE] --> CUT[Connection cut]
  EXIT --> CUT
  CUT --> MISS[Response or completion event missing]
  MISS --> UI[Viewer state stale]
  MISS --> RETRY[Client retries or duplicate intent]
~~~

安全な変更境界は、server停止受付、deadline付きdrain、background stop、store/provider close、process exitの順序を明示すること。SSEの長寿命接続は無期限待機させず、deadlineと切断eventの契約が必要である。

### INCONSIST-001 Route ownership移行の影響伝播

~~~mermaid
flowchart TD
  OPS[Ops registrar] --> REV[Revenue routes]
  OPS --> ADV[Advisor routes]
  OPS --> AGT[Agent profile routes]
  OPS --> SCH[Scheduler routes]
  OPS --> WS[Workstream routes]
  MOVE[Move one route] --> DUP{Old registration remains}
  DUP -->|yes| PANIC[Duplicate handler or shadowed owner]
  DUP -->|no| TEST[Contract test]
~~~

一括移動せず、path一覧をsnapshot/contract testで固定し、一featureごとに旧登録削除と新登録追加を同じdiffで行う。

### TEST-001 Remote agent test不足の影響伝播

~~~mermaid
flowchart TD
  FLAG[Flag or config change] --> PARSE[CLI parse]
  PARSE --> CONNECT[Remote connection]
  CONNECT --> EXEC[Remote job]
  EXEC --> EXIT[Exit code and logs]
  NTEST[No package contract test] --> MISS[Regression not isolated]
  MISS --> PROD[Failure appears only in integration or operation]
~~~

最初のtestはpure parser/config、次にfake transport、最後にbinary-level e2eと段階化し、外部runtimeをunit testへ持ち込まない。

### DEV-001 旧path参照の影響伝播

~~~mermaid
flowchart TD
  SPEC[Current spec] --> OLD[pkg agent old path]
  OLD --> SEARCH[Reader searches missing file]
  SEARCH --> GUESS[Guesses replacement]
  GUESS --> WRONG[Edits wrong layer]
  WRONG --> BOUND[Responsibility boundary regression]
~~~

単純な文字列置換ではなく、旧`pkg/agent`責務が現在のdomain/application/infrastructureのどこへ分割されたかを対応表で示す必要がある。

## 優先対応マトリクス

| 優先度 | 件数 | 対象 | 判断 |
| --- | ---: | --- | --- |
| 即時設計 | 1 | ERR-001 | restart時のruntime整合性に直結 |
| 計画対応 | 3 | INCONSIST-001、TEST-001、DEV-001 | contractを先に固定して順次修正 |
| 改善機会 | 2 | NI-001、DEV-002 | 移行進捗と文書品質の改善 |

### 推奨実施順

1. ERR-001をshutdown contract test付きで設計する。
2. feature route path contractを固定し、INCONSIST-001とNI-001を同じmigration系列で縮小する。
3. `cmd/rencrow-agent`のpure contract testを追加する。
4. current正本の旧pathとGo version表記を修正する。

## 対応しない場合のリスク

- ERR-001: restart時にrequest、SSE event、永続化完了の一部が欠落し、client側の再試行で重複intentが生じ得る。
- INCONSIST-001/NI-001: feature追加のたびにOpsと個別registrarの双方を調べる必要があり、二重登録のriskが増える。
- TEST-001: remote agent固有の回帰が通常のpackage testで局所化されない。
- DEV-001/DEV-002: 新規参加者や自動化が誤ったpath/toolchainを前提にする。
