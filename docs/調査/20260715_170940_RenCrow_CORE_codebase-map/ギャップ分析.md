## エラー処理欠落（ERR）

### ERR-001 [中] signal shutdownがHTTP requestをdrainしない

- モジュール: cmd_runtime。
- 場所: `cmd/rencrow/main.go:89-98`、`cmd/rencrow/main.go:179-195`。
- 証拠: signal goroutineは`dependencies.Shutdown()`と`os.Exit(0)`を呼ぶが、`server.Shutdown`呼び出しはない。
- 期待: SIGINT/SIGTERM時、new requestを止め、in-flight requestを期限付きでdrainしてから依存資源を閉じる。
- 実際: signal goroutineは`Dependencies.Shutdown()`後に`os.Exit(0)`を呼び、作成した`http.Server`へ`Shutdown(ctx)`を呼ばない。
- 影響: 応答途中のViewer/API requestやSSE更新が切断され、job完了event・永続化・client ACKが欠落する可能性がある。
- 反証条件: process managerが必ずtrafficを先にdrainする、または全handlerが中断安全であることを示す運用・test証拠がある。
- 推奨: signalをmain control flowへ戻し、`server.Shutdown(timeoutCtx)`、background stop、dependency closeの順序を明文化してtestする。
- 履歴判定: long_standing。現構造の導入以降に明示drain追加は確認できない。
