## 4. 見送りまたは用途限定

### 4.1 headroom

Repository:

```text
https://github.com/headroomlabs-ai/headroom
```

評価:

```text
context compression / proxy / MCP / CLI wrap として用途はある。
ただし `headroom-ai[all]` は約 5.9GB と重く、RenCrow_Tools から削除済み。
再検証する場合は `[all]` ではなく、proxy / mcp / code など必要 extras を絞って導入する。
Claude / Codex の global wrap は明示承認なしに行わない。
```

### 4.2 SkillSpector

Repository:

```text
https://github.com/NVIDIA/SkillSpector
```

評価:

```text
AI agent skill の security scanner として有望。
今回の導入対象には含めないが、外部 skill 導入前 gate として次の有力候補。
release package が整うまでは local scan / CI proof-of-concept から始める。
```

### 4.3 Agent-Reach

Repository:

```text
https://github.com/Panniantong/agent-reach
```

評価:

```text
capability router / health checker / primary-fallback backend 選定の参考として有用。
ただし、RenCrow では Source Registry / artifact / audit log / Human approval と接続した read-only research router を RenCrow_Tools に自作する方針とする。
Agent-Reach CLI、wrapper、~/.agent-reach、~/.agents/skills/agent-reach、~/.claude/skills/agent-reach は削除済み。
yt-dlp は自作 router でも使うため残す。
```

取り込むべき本質:

```text
- platform ごとの primary / fallback backend
- backend ごとの health check
- read-only と account operation の分離
- cookie / login が必要な経路の明示 approval
- source_url / retrieved_at / tool / exit_code / raw_artifact / summary_artifact の保存
```

### 4.4 LMCache

Repository:

```text
https://github.com/LMCache/LMCache
```

評価:

```text
vLLM / SGLang などの serving layer を使う場合に効く KV cache layer。
現状の RenCrow では infra 変更が大きいため、RenCrow_LLM が vLLM 系へ寄るまで仕様参考に留める。
```

### 4.5 OpenMontage

```text
https://github.com/calesthio/OpenMontage
```

評価:


## 5. 次の候補

今回導入した 8 の次に検討するなら、優先度は次とする。

```text
1. SkillSpector
   外部 skill / Claude skill / Codex skill の安全 scan gate。

2. agent-skills
   RenCrow skill 設計の reference import。

3. PaddleOCR
   document ingestion が必要になった段階で RenCrow_Tools へ隔離導入。
```

## 6. 運用方針

```text
- 導入済みツールは RenCrow_Tools の wrapper から使う。
- login / cookie / account が必要な platform は Human approval 必須。
- session history / token usage / prompt logs は sensitive data として扱う。
- compression / proxy / MCP routing は opt-in にする。
- 参考 repo は直接移植せず、仕様、skill、runbook、approval gate へ変換して取り込む。
```
