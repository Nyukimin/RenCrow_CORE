# 応対抽出 実装項目 2026-05-14 E2E

- `/viewer/send` は複雑な自然文リクエストを受理し、job を生成できること。
- 明示指定された `/ops` 入力は `OPS` route として処理すること。
- `OPS` route の job は Worker / Shiro により実行し、応答本文を返せること。
- job 完了時に execution evidence を保存し、成功時は `status=passed` として記録すること。
- execution evidence には `attempt_count` と `repair_count` を記録し、job 単位で確認できること。
- LLM が `parse_status=no_reasoning` 等で内部検討文を本文へ混入させた場合でも、RenCrow 側の表示本文には内部検討文を出さないこと。
- `Final answer:` / `Final Answer:` 以降の最終回答本文を抽出し、ユーザー向け表示本文として扱えること。
- 複雑文の要約要求では、日本語一文として成立する最終回答を返せること。
- `evidence.status=passed` の場合、Viewer の job detail / list も `completed` 扱いにすること。
- job event に `entry.stage=completed` がある場合も、Viewer 表示用 `status` を `completed` に倒すこと。
- `mio_reported=false` は「最終報告イベント未記録」として別表示にし、実行完了状態とは混ぜないこと。
