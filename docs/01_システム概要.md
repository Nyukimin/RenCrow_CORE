# システム概要

RenCrow_CORE は、会話型インターフェースと作業実行を、安全な複数エージェント構成で統合する orchestration runtime です。

## 目的

- 人格を保った自然な対話と、明示的な作業依頼を同じ入口で扱う。
- 会話、判断、提案、実行、承認の責任者を分離する。
- 会話履歴と Knowledge を provenance 付きで Recall する。
- 長時間の作業を Workstream、Scheduler、Heartbeat で継続する。
- 実行状態、失敗、degraded状態をDebug Viewerから観測し、外部利用者にはRenCrow_PORTALで許可した状態だけを公開する。

## CORE が所有する範囲

- RenCrow_PORTAL、Debug Viewer、LINE、Slack、Discord、Telegramなどの入力契約
- Mio による通常メッセージの route decision と明示 command の優先
- Agent、Advisor、Tool の契約と責務境界
- Persona、会話履歴、RecallPack、Knowledge Relation、trace
- side effect 前の approval gate と監査記録
- Workstream、Backlog、Scheduler、Heartbeat、Economic Objective
- Viewer REST/SSE と外部 module の状態 projection

## 外部モジュールが所有する範囲

| モジュール | 所有する実装本体 |
| --- | --- |
| RenCrow_LLM | 推論 server、model 定義、sampling、management runtime |
| RenCrow_STT | 音声認識処理 |
| RenCrow_TTS | 音声合成処理 |
| RenCrow_Vision | 画像・映像認識処理 |
| RenCrow_GAMES | ゲーム世界、ActionIntent、Executor、Replay、Observer UI |
| RenCrow_Tools | 横断 tool、browser sidecar、data converter、validation CLI |
| RenCrow_PORTAL | 外部利用者向けview/lab Web UI、許可制API proxy |

CORE はこれらへの接続契約、選択、health、状態表示を担います。外部 module が未起動でも CORE の実装欠落とは限らず、利用時には明示的な unavailable/degraded として扱います。
